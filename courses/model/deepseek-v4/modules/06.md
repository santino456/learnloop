## [m6-why-muon] 为什么用 Muon

DeepSeek-V4 系列把 Muon 优化器用于大部分模块，原因是：

- 收敛速度更快。
- 训练稳定性更好。

但 Muon 并不适用于所有参数。V4 仍保留 AdamW 用于：

- Embedding 模块
- Prediction head 模块
- mHC 的静态偏置和门控因子
- 所有 RMSNorm 的权重

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 2.4 节。

## [m6-algorithm] Muon 算法概览

对每个逻辑上独立的权重矩阵 W ∈ R^(n×m)：

1. 计算梯度 G_t = ∇_W L_t(W_{t-1})。
2. 更新动量 buffer：M_t = μ M_{t-1} + G_t。
3. 使用 Nesterov trick：对 μ M_t + G_t 做正交化，得到 O'_t。
4. 按矩阵维度缩放更新：O_t = O'_t · √(max(n, m)) · γ。
5. 执行权重衰减和参数更新：W_t = W_{t-1} · (1 - ηλ) - η O_t。

关键在第 3 步：通过正交化让更新矩阵接近正交矩阵，从而控制更新方向的质量。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，Algorithm 1。

## [m6-newton-schulz] 混合 Newton-Schulz 迭代

正交化的目标：给定 M 的 SVD 分解 M = U Σ V^T，希望逼近 U V^T。

Newton-Schulz 迭代先对 M 做 Frobenius 范数归一化，保证最大奇异值 ≤ 1，然后迭代：

```
M_k = a M_{k-1} + b (M_{k-1} M_{k-1}^T) M_{k-1} + c (M_{k-1} M_{k-1}^T)^2 M_{k-1}
```

DeepSeek-V4 使用**两阶段混合迭代**，共 10 步：

- 前 8 步：系数 (a, b, c) = (3.4445, -4.7750, 2.0315)，快速收敛。
- 后 2 步：系数 (a, b, c) = (2, -1.5, 0.5)，稳定奇异值到 1。

::: checkpoint
混合 Newton-Schulz 的两阶段各自作用是什么？

--- answer
前 8 步快速把奇异值拉近 1；后 2 步精确稳定到 1。
---
:::

## [m6-no-qk-clip] 为什么不用 QK-Clip

在 DeepSeek-V3 的训练中，Muon 通常需要配合 QK-Clip 来防止 attention logits 爆炸。但 V4 的注意力架构本身允许直接对 query 和 KV 条目做 RMSNorm，因此 attention logits 不会爆炸，也就不需要 QK-Clip。

::: decision
如果你要在自己的模型中使用 Muon，是否一定需要 QK-Clip？

- A. 一定需要，Muon 本身会导致 attention logits 爆炸。
- B. 不一定，如果注意力架构本身做了 query/KV 归一化，可以避免爆炸。
- C. 只有在大模型中才需要。

--- perspective
依据：DeepSeek-V4 论文第 2.4 节。

B 更合理。是否需要 QK-Clip 取决于注意力架构是否能有效控制 logits 幅度；不能把它当作 Muon 的固定配套。
---
:::

## [m6-summary] 模块小结

- Muon 通过正交化权重更新来提升收敛和稳定性。
- DeepSeek-V4 使用 10 步混合 Newton-Schulz 迭代做正交化。
- 由于注意力架构自带 query/KV RMSNorm，V4 不需要 QK-Clip。
- Embedding、head、mHC 静态偏置和 RMSNorm 权重仍用 AdamW。
