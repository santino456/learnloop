## [m6-goal] 目标与准备

最小的本地客户端只需要做四件事：启动 Agent 进程、发送 initialize、发送 prompt、读取响应。

本模块用伪代码风格示例。在真正运行前，务必对照你当前使用的 Agent 实现确认方法名、字段和协议版本。

## [m6-process] 启动子进程

```python
import json
import subprocess

proc = subprocess.Popen(
    ["example-agent", "acp"],
    stdin=subprocess.PIPE,
    stdout=subprocess.PIPE,
    stderr=subprocess.PIPE,
    text=True,
    bufsize=1,
)
```

这里用 `example-agent` 代替真实的私有配置。测试时请替换成你实际要测的 Agent。

::: checkpoint
`stderr=subprocess.PIPE` 和 `stdout=subprocess.PIPE` 为什么要分开捕获？

--- answer
stdout 必须保持纯 JSON 行以便解析；stderr 用于日志和错误输出，两者不能混合。
---
:::

## [m6-send-receive] 发送与接收 JSON 行

```python
def send(message):
    proc.stdin.write(json.dumps(message) + "\n")
    proc.stdin.flush()

def receive():
    line = proc.stdout.readline()
    if not line:
        raise RuntimeError("agent closed stdout")
    return json.loads(line)
```

换行和刷新不是装饰。没有它们，Agent 可能永远等不到完整输入。

## [m6-prompt] 发送一条提示

```python
send({
    "jsonrpc": "2.0",
    "id": 2,
    "method": "prompt",
    "params": {
        "messages": [
            {"role": "user", "content": [{"type": "text", "text": "用一句话解释 ACP。"}]}
        ]
    }
})
print(receive())
```

> 验证提示：这只是教学骨架。在把它当作可执行代码之前，先运行真实 Agent 的帮助命令或文档检查。

## [m6-buffering] 缓冲规则

- 如果协议发送的是文本 JSON，使用文本模式。
- 每行写一个完整 JSON 对象。
- 每次请求后显式刷新。
- 保持 stdout 干净，日志发到 stderr。

::: checkpoint
如果 `send` 里漏掉了 `flush()`，程序可能出现什么现象？

--- answer
消息可能滞留在缓冲区，Agent 永远读不到完整 JSON 行，导致客户端挂起。
---
:::

## [m6-summary] 模块小结

- 最小本地客户端依赖子进程管道。
- 行分隔 JSON 要求严格的换行和刷新行为。
- 可运行示例必须针对真实 Agent 版本验证。

```python
proc = subprocess.Popen(["agent"], stdin=subprocess.PIPE, stdout=subprocess.PIPE, text=True)

def send(message):
    proc.stdin.write(json.dumps(message) + "\n")

send({"jsonrpc":"2.0","id":1,"method":"initialize","params":{}})
```

--- answer
`send` 缺少 `proc.stdin.flush()`。修复后：

```python
def send(message):
    proc.stdin.write(json.dumps(message) + "\n")
    proc.stdin.flush()
```
