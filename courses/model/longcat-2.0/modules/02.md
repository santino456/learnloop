## [m2-motivation] 为什么智能体需要稀疏注意力

::: concept
title: 长输入瓶颈
why: 智能体工作流会把代码库、多轮对话历史、工具返回结果等长文本塞进上下文。

智能体应用正在把 LLM 推向高效长输入处理。稠密全注意力的计算量随序列长度平方增长，100 万 token 的推理成本极高。稀疏注意力通过只选择部分 token 参与注意力，来降低这一成本。

延伸阅读：
- [Efficient Transformers: A Survey](https://arxiv.org/abs/2009.06732)（Transformer 与高效注意力机制综述，适合理解稀疏注意力的动机）
- [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)（本章 LSA 部分的主要来源）
:::

## [m2-dsa] 起点：DeepSeek Sparse Attention

::: concept
title: DSA 的设计与瓶颈
why: LSA 是 DSA 的演进，先理解基线再理解改进。

DeepSeek Sparse Attention（DSA）是 DeepSeek-V3.2 提出的细粒度稀疏注意力机制，核心组件包括：

- **Fine-grained sparse attention**：以 token 级或块级稀疏模式替代全量 attention。
- **Lightning Indexer**：动态决定每个 query 应该 attend 哪些 key/value，是 DSA 的关键路径。

DSA 在长上下文训练和推理上显著降低了计算量，但 LongCat 团队分析发现 Lightning Indexer 仍是主要瓶颈：

1. **输出不连续** —— 选中的 key/value 在内存中不连续，导致 HBM 访问碎片化。
2. **打分成本二次增长** —— 细粒度候选打分需要在大量候选上做精确比较。

Source:
- [DeepSeek-V3.2 Technical Report](https://arxiv.org/abs/2512.02556)（DSA 与 Lightning Indexer 的原始出处）
- [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)（LSA 在 DSA 基础上的改进）
:::

## [m2-lsa-components] LongCat 的三项正交改进

::: concept
title: Streaming-aware Indexing（SI）
why: 内存访问模式决定了加速器上的有效带宽。

SI 重新塑造 token 选择预算，把硬件友好的连续访问与动态随机选择结合起来。它将碎片化的内存访问转化为可预测的顺序读取，从而实现合并的 HBM 访问和更高的有效带宽。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: concept
title: Cross-Layer Indexing（CLI）
why: 相邻层的注意力显著性在经验上是稳定的。

CLI 利用相邻层注意力显著性的稳定性来均摊索引成本：推理时一次索引可为连续多层服务。这通过训练时的跨层蒸馏实现。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: concept
title: Hierarchical Indexing（HI）
why: 对完整上下文做细粒度打分代价太高。

HI 采用由粗到细的两阶段方案：

1. **粗召回**：通过块级近似打分快速筛选候选块。
2. **细打分**：在召回的候选块内部做细粒度 token 选择。

这样大幅缩小了 indexer 每次查询需要处理的候选空间。在 LongCat-2.0 中，HI 以 training-free 方式启用，并仅用于选定的超长上下文任务。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

## [m2-mtp] 把 LSA 扩展到投机解码

::: concept
title: LSA + 3 步 Multi-Token Prediction
why: 投机解码需要便宜的 draft token，但不能增加显存压力。

LSA 的三项策略被扩展到一个 3 步 MTP 模块：

- 在 **target 模型** 中，每两个连续层共享一次索引。
- 在 **draft 模型** 中，三个 MTP draft 步共享一次索引，第 2、3 步复用第 1 步生成的索引集合。

延伸阅读：[A Comprehensive Survey of Speculative Decoding](https://arxiv.org/abs/2401.07851)（投机解码综述，了解 draft model 如何加速 LLM 解码）

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: compare
left: 稠密注意力
right: LongCat 稀疏注意力

- 成本增长 | 随序列长度平方增长 | 通过 token 选择实现次平方增长
- 内存访问 | 量大但规则 | 基线稀疏方案碎片化；SI 后合并访问
- 层间索引 | 每层重新计算 | 通过 CLI 跨层共享
- 候选打分 | 全量细粒度扫描 | HI 粗到细筛选
:::
