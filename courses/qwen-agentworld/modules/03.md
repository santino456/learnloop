## [m3-recipe] Three-Stage Training Recipe

Qwen-AgentWorld is trained end-to-end with environment modeling as the explicit objective from continual pre-training onward. The principle is "CPT injects, SFT activates, RL sharpens."

::: figure
src: assets/qwen-agentworld-fig-5.png
alt: Figure 5 showing the three-stage training pipeline of Qwen-AgentWorld: CPT injects world knowledge, SFT instills next-state-prediction thinking patterns, RL sharpens output quality.
caption: Figure 5: Three-stage training pipeline of Qwen-AgentWorld. Stage 1 CPT injects world knowledge; Stage 2 SFT instills next-state-prediction thinking patterns; Stage 3 RL sharpens output quality.
:::

::: flow
Stage 1: CPT (Continual Pre-Training)
  → Injects world knowledge from 10M+ environment interaction trajectories
  → Plus specialized-domain corpora (industrial control, cybersecurity, law, medicine, finance, current affairs)
  → Uses standard next-token prediction with information-theoretic loss masking

Stage 2: SFT (Supervised Fine-Tuning)
  → Activates next-state prediction as an explicit reasoning pattern
  → 7,094 trajectories (69.2% retention from 10,250 candidates after rejection sampling)
  → Prompt template diversification + rejection sampling

Stage 3: RL (Reinforcement Learning)
  → Sharpens simulation fidelity with hybrid rubric-and-rule rewards
  → Uses GSPO (Group Sequence Policy Optimization)
  → 9:1 rubric-to-rule reward ratio
:::

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3

## [m3-cpt] Stage 1: Continual Pre-Training with Loss Masking

CPT trains under the standard next-token prediction objective. Multi-turn environment trajectories are framed as world-modeling tasks: the system prompt defines the simulation context, user turns carry the agent’s actions, and assistant turns carry the environment’s responses.

::: concept
title: Information-theoretic loss masking

**Information-theoretic loss masking**: Many turns in tool-use trajectories are boilerplate (tools that echo input, APIs that mirror request parameters). The gradients induced by these turns are low-quality and introduce noise, but the turns cannot simply be removed because subsequent turns depend on them as context.

To address this, four statistics are computed per (action, observation) pair:
- **OL** (Overlap) = | W_act ∩ W_obs | / |W_act|
- **Nov** (Novelty) = | W_obs \\ W_act | / |W_obs|
- **Jac** (Jaccard) = | W_act ∩ W_obs | / |W_act ∪ W_obs|
- **R** (length ratio) = | obs | / |act|

Each turn is assigned to one of seven semantic categories with a keep ratio (fraction of tokens used in loss computation).

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.2
:::

::: figure
src: assets/qwen-agentworld-table-3.png
alt: Table 3 showing seven turn categories for information-theoretic loss masking with statistical signatures, intuition, and keep ratios.
caption: Table 3: Seven turn categories for information-theoretic loss masking. Categories are determined from statistical signals rather than tool names. Keep ratio is the fraction of tokens used in loss computation.
:::

::: concept
title: Key insight

**Key insight**: Categories are determined statistically rather than by tool name, keeping the classification tool-agnostic. Masked turns are excluded from the loss computation while their tokens are retained as context for subsequent turns. This decouples "learning the next state" from "learning the next token": loss is computed only on turns that carry genuine environment information.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.2
:::

## [m3-sft] Stage 2: Supervised Fine-Tuning

SFT shifts from the non-thinking regime of CPT to thinking trajectories that contain explicit reasoning chains. The model learns to engage in next-state prediction explicitly: identifying what an action requests, recalling the prior state, and anticipating the expected response format.

::: figure
src: assets/qwen-agentworld-table-4.png
alt: Table 4 showing rejection sampling statistics per domain with candidates, retain rate, and final SFT counts.
caption: Table 4: Rejection sampling statistics per domain. "Candidates" is the number of queries with complete rollouts. "Retain rate" is the fraction of queries whose best-of-three trajectory exceeds the quality threshold. "Final SFT" is the count after filtering.
:::

::: concept
title: Rejection sampling

**Rejection sampling**: For each query, three rollouts are generated from a general-purpose reasoning model. The candidates are scored by an independent judge and compared pairwise to select the highest-quality trajectory. If the winning trajectory’s score falls below the minimum threshold, the query is discarded. Starting from 10,250 candidate queries, the pipeline retains 7,094 trajectories (69.2% retention rate).

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.3
:::

::: concept
title: Prompt template diversification

**Prompt template diversification**: Each SFT sample has its default system prompt replaced by one uniformly sampled from 10 template variants. The trajectory’s dynamic content (tool definitions, demonstrations, simulation instruction) is preserved and re-injected into the new template. This improves the model’s generalization across system prompt variations.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.3
:::

## [m3-rl] Stage 3: Reinforcement Learning

RL for LWM poses distinctive challenges due to the open-ended nature of environment feedback prediction and extreme prompt–output asymmetry: the prompt often extends to tens of thousands of tokens, whereas the output typically contains only a few hundred to a few thousand tokens.

::: concept
title: GSPO (Group Sequence Policy Optimization)

**GSPO (Group Sequence Policy Optimization)**: The RL algorithm used for LWM training. The prompt is capped at 128k tokens for the RL pool.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4
:::

### [m3-reward] Reward Design

The reward combines two complementary signals at a 9:1 ratio (rubric:rule).

::: concept
title: Five-dimensional rubric (LLM judge)

**Five-dimensional rubric (LLM judge)**: Each predicted observation is scored by an LLM judge on five dimensions (Format, Factuality, Consistency, Realism, Quality), each on a 1–5 scale. The total reward equals the mean × 5, yielding a range of [5, 25]. Each domain uses a tailored judge system prompt with content-type classification to reduce false negatives from irreproducible details.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4.1
:::

::: concept
title: Rule-based verifier

**Rule-based verifier**: A subset of the data carries executable verifier code that produces a binary 0/1 correctness signal, scaled to [0, 25] to align with the rubric’s range. Rule-based rewards serve as an objective anchor, mitigating reward hacking induced by open-ended rewards.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4.1
:::

::: decision
**When to use rubric vs. rule-based rewards**

--- answer

**Use rubric rewards when:**
- The observation space is open-ended and no executable ground truth exists.
- Multi-dimensional feedback is needed to distinguish between format errors, factual errors, and consistency issues.

**Use rule-based rewards when:**
- Executable verifiers can be constructed for deterministic checks (e.g., exact file contents, JSON schema compliance).
- You need an objective anchor to prevent reward hacking from judge bias.

**Best practice**: Combine both at a 9:1 ratio (rubric:rule) to balance multi-dimensional feedback with strict binary correctness.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4.1
---
:::

## [m3-stability] Training Stability

Three failure modes were identified through systematic ablation.

::: concept
title: Reward collapse from multi-turn expansion

**Reward collapse from multi-turn expansion**: When training trajectories are expanded into multiple samples, the resulting instances share a long common prefix, causing training to collapse quickly (related to the "Echo Trap"). **Solution**: Restrict expansion to exactly one turn per trajectory in the RL pool, giving each training sample a unique prediction target with no shared-prefix overlap.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4.2
:::

::: concept
title: Reward hacking through self-praise

**Reward hacking through self-praise**: The policy learns to embed qualitative affirmations or echo key terms from the judge prompt to inflate scores without improving simulation fidelity. **Mitigations**: (1) rule-based verifier provides a stable anchor; (2) content-type classification narrows rubric scope; (3) strict tag extraction separates thinking block from predicted observation so self-praise never reaches the judge.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §3.4.2
:::

**Module Summary**: The three-stage recipe (CPT → SFT → RL) progressively injects environment knowledge, activates next-state-prediction reasoning, and sharpens simulation fidelity.
