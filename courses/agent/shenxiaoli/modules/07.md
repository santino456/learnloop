## [m7-topology] 部署拓扑

```
用户浏览器
    │
    ▼
Nginx (:443 HTTPS, Certbot 自动续签)
    ├── /api/*        → proxy_pass → FastAPI (:8000, 2 uvicorn worker)
    ├── /kb/campus/images/* → proxy_pass → OSS 内网端点 (本地开发回退)
    ├── /skills/*     → proxy_pass → OSS 内网端点
    ├── /assets/*     → 静态文件, 1年缓存 (Vite hash 文件名)
    └── /*            → frontend/dist/index.html (SPA fallback)
                            │
                    OSS 公网端点 (图片直连, Gbps 带宽)
                            │
                    PostgreSQL 14 (本地 :5432)
```

核心设计：**ECS 只跑 API 和静态文件；图片流量走 OSS 公网，不占 ECS 带宽。**

## [m7-nginx] Nginx 关键配置

### SSE 专用代理

```nginx
location /api/chat/stream {
    proxy_pass http://127.0.0.1:8000;
    proxy_buffering off;        # 必须关缓冲
    gzip off;                   # 必须关 gzip
    proxy_read_timeout 300s;    # 长连接超时

    add_header Cache-Control "no-store";
    add_header Content-Type "text/event-stream";
}
```

三个关键点：
- `proxy_buffering off` — 不缓冲响应。SSE 需要逐事件推送，缓冲会延迟所有事件直到连接关闭。
- `gzip off` — gzip 需要在缓冲区积累数据才能压缩，会导致 SSE 数据堆积。
- `proxy_read_timeout 300s` — SSE 保持连接最长达 5 分钟。

### SPA 回退

```nginx
location / {
    try_files $uri /index.html;
    # index.html 不缓存（防止新版上线后用户拿到旧版）
    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
}
```

### 静态资源缓存

```nginx
location /assets/ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

Vite 构建产物的文件名带 hash（如 `index-abc123.js`），文件内容变化 → hash 变化 → 新 URL，所以可以安全地缓存一年。

## [m7-deploy] 一键部署脚本

`deploy.sh` 是全链路部署的总入口：

```bash
#!/bin/bash
BRANCH=${1:-refactor}

# 1. 拉取最新代码
git pull origin "$BRANCH"

# 2. 后端依赖
source backend/.venv/bin/activate
pip install -r backend/requirements.txt

# 3. 前端构建
cd frontend && npm run build && cd ..

# 4. 重启服务
sudo systemctl restart shenxiaoli

# 5. 健康检查（最多 30 次，1 秒间隔）
for i in {1..30}; do
    if curl -sf http://127.0.0.1:8000/health > /dev/null; then
        echo "Deploy OK"
        exit 0
    fi
    sleep 1
done
echo "Deploy FAILED - health check timeout"
exit 1
```

完整部署时间：约 20-30 秒（git pull 2s + pip install 3s + npm build 12s + systemctl restart 3s + health check 2s）。

## [m7-oss] OSS 配置要点

### 防盗链

```json
{
    "Referer": {
        "AllowEmptyReferer": false,
        "AllowReferer": ["https://shenxiaoli.cn", "https://www.shenxiaoli.cn"],
        "RefererPolicy": "white"
    }
}
```

- 仅允许 `shenxiaoli.cn` 域名的 Referer
- 禁止空 Referer（防止 curl 直接下载图片）
- 图片被其他网站引用时返回 403

### WebP 压缩

所有新增 Skill 图片需生成 WebP 版本：

```bash
for f in $(find <目录>/ -type f \( -name '*.jpg' -o -name '*.png' \)); do
    webp="${f%.*}.webp"
    [ -f "$webp" ] || cwebp -q 82 "$f" -o "$webp"
done
```

上传到 OSS 时原图（jpg/png）和 WebP 并存，浏览器优先选 WebP。

## [m7-systemd] systemd 服务

```ini
[Unit]
Description=ShenXiaoLi API Server
After=network.target postgresql.service

[Service]
User=hqyone
WorkingDirectory=/home/hqyone/projects/shenxiaoli/backend
ExecStart=/home/hqyone/projects/shenxiaoli/backend/.venv/bin/python \
    -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 2
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

关键设计：
- `--workers 2` — 2C4G 机器上 2 个 worker 能并发处理 4 个聊天（每个 worker 的 `chat_semaphore=4`）
- `Restart=always` — 崩溃自动重启
- `RestartSec=5` — 重试间隔 5 秒，防止频繁重启打满 CPU
- 监听 `127.0.0.1` — 不暴露到公网，只让 Nginx 代理

## [m7-monitoring] 运维命令速查

```bash
# 服务状态
sudo systemctl status shenxiaoli

# 实时日志
sudo journalctl -u shenxiaoli -f

# 查看最近 100 行
sudo journalctl -u shenxiaoli -n 100 --no-pager

# Nginx 配置测试
sudo nginx -t

# Nginx 平滑重载
sudo nginx -s reload

# 健康检查
curl http://127.0.0.1:8000/health

# 检查端口占用
ss -tlnp | grep -E ':(80|443|8000) '
```

## [m7-limitations] 已知短板

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 重启有 3-5 秒停机窗口 | 期间用户收不到响应 | 用户量小时可接受 |
| 速率限制不跨 worker | 2 worker 各算各的 | 当前 2 worker 影响可控 |
| 没有 CI/CD | 靠手动 `./deploy.sh` | 部署频率低，手动可以 |
| 没有自动备份 | 需手动 `pg_dump` | 需补充 cron job |
| 日志只在 journald | 无持久化分析 | 当前凭感觉看就行 |

## [m7-summary] 模块小结

- 裸机部署拓扑：Nginx → FastAPI + OSS 直连 + PostgreSQL。
- SSE 需要三个 Nginx 关键配置：buffering off / gzip off / read_timeout 300s。
- 部署脚本 5 步：pull → pip → build → restart → health check，约 30 秒。
- OSS 防盗链只允许 `shenxiaoli.cn`，禁止空 Referer。
- 已知短板都在可控范围内，用户量增长后有明确的升级路径。
