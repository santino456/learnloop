## [m6-foundation] World Model as Agent Foundation Model

In this application, the same model that selects actions (agent) also predicts environment states (world model). LWM training enables the agent to mentally simulate the consequences of a candidate action before committing, effectively using world modeling as an internal planning step that improves action quality.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.2

## [m6-warmup] LWM RL Warm-Up and Cross-Task Transfer

LWM RL warm-up familiarizes the agent with how environment states evolve in response to user actions: how different tool calls and search queries yield different responses, which tools are more effective, and how state transitions propagate across turns. This amounts to learning next-state prediction as a meta-reasoning pattern.

**Radical cross-task generalization**: LWM RL trains on single-turn, non-agentic trajectories with no tool calls, yet transfers to multi-turn, tool-calling agentic tasks across seven benchmarks. ## [m6-mechanism] Prediction-Driven Action Refinement

The mechanism behind the transfer is that the RL-trained model systematically performs mental simulation of environment responses before executing actions, using its internalized world model to predict outcomes, identify infeasible approaches, and refine its action plan—all within the thinking trace and before any real execution.

**Prediction accuracy improvement**: RL training improves prediction accuracy from 69.9% to 78.3% (+8.4%), confirming that the model’s internal world model becomes more faithful through LWM training. ::: concept
title: Prediction-driven action refinement

**Prediction-driven action refinement**: On the mailman task, both models encounter the same Postfix recipient-rejection error. The model after LWM RL correctly predicts that configuring transport_maps alone will not work because Postfix rejects unknown recipients before consulting transport routing, enabling it to refine its action toward modifying local_recipient_maps. The model before LWM RL incorrectly predicts that transport routing precedes recipient validation, leading to futile exploration before timing out.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.2
:::

## [m6-exercise] Exercise: Quantify Prediction Accuracy

::: exercise
**Task**: Design a protocol to measure prediction accuracy in an agent’s thinking traces.

**Requirements**:
1. Define what counts as an "explicit prediction" in a thinking trace (e.g., phrases like "If I run X, the environment will respond with Y").
2. Specify the semantic consistency criterion for marking a prediction as correct.
3. Describe how to handle predictions that are partially correct or ambiguous.
4. Propose a baseline and an intervention (e.g., before vs. after LWM RL) and the expected effect size based on the paper’s results (+8.4% accuracy improvement).

--- answer
**Verification criteria**: The protocol should be reproducible across different domains and should not require access to the model’s internal activations—only the thinking trace and the ground-truth environment observation.

A complete answer should include:
- Explicit prediction marker: a sentence in the thinking trace that states an anticipated environment response before the action is committed.
- Semantic consistency: compare the predicted response to the actual observation; allow partial credit only when the predicted and actual responses differ in surface form but agree on all actionable facts.
- Baseline vs. intervention: measure accuracy on the same benchmark before LWM RL warm-up and after; expect an improvement comparable to the paper’s +8.4% (69.9% → 78.3%).
---

:::
