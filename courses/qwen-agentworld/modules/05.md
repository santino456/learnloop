## [m5-simulator] World Model as Environment Simulator

In this application, Qwen-AgentWorld serves as a standalone environment simulator: the policy agent and the world model are separate models. The simulator provides two properties that real environments do not: scalability and controllability.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1

## [m5-controllable] Controllable Simulation

Controllable simulation uses natural-language instructions to shape how Qwen-AgentWorld behaves during training, at both the trajectory and turn levels.

::: concept
title: Environment adaptation

**Environment adaptation**: The simulator’s behavior and style are adjusted through instructions to inject targeted perturbations, thereby systematically exposing agent weaknesses. Control instructions modify the simulation style and content to produce conditions more extreme or targeted than what real deployments provide.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2
:::

## [m5-fictional] Fictional-World Construction

::: concept
title: Fictional-world construction

**Fictional-world construction**: The simulator generates an entirely fictional environment that is structurally realistic yet factually disjoint from the real world. For Search, Qwen-AgentWorld simulates a fully fictional, self-consistent world from a compact initial specification. This offers two structural advantages: (1) the agent cannot bypass the search tool by answering from parametric memory; (2) the agent cannot confuse training-time search results with real-world knowledge.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2
:::

::: decision
**When to use decoupled simulator vs. real environment**

--- perspective

**Use decoupled simulator (Sim RL) when:**
- You need to scale to thousands of parallel environments without real infrastructure.
- You want to inject targeted perturbations or construct fictional worlds for adversarial training.
- Real environment access is expensive, slow, or infeasible (irreversible operations, proprietary deployments).

**Use real environment (Real RL) when:**
- You need exact fidelity for safety-critical or high-stakes domains.
- The environment is simple and fast enough that scaling is not a bottleneck.
- You are in the final validation phase and need to confirm sim-to-real transfer.

**Important caveat**: Sim RL effectiveness depends on providing the world model with a sufficiently detailed initial state; without it, simulation fidelity degrades and downstream gains diminish.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1
---
:::

## [m5-exercise] Exercise: Design a Controllable Simulation

::: exercise
**Task**: Design a controllable simulation instruction for a Terminal-domain training scenario.

**Scenario**: You want to train an agent to handle disk-space errors gracefully during package installation.

**Requirements**:
1. Write a simulation instruction that specifies the initial state (OS, disk space, installed packages).
2. Specify the controllable perturbation: the simulator should simulate a "no space left on device" error during pip install, but only after the download phase completes normally.
3. Specify the expected agent behavior: the agent should detect the error, clean the pip cache, and retry with --no-cache-dir.

--- answer
**Verification criteria**: The simulation instruction must make the intended response boundary explicit to reduce hallucination during training. The initial state must be sufficiently detailed to constrain the expected observation.

A complete answer should include:
- Initial state: Ubuntu 22.04, Python 3.10, pip 23.x, 2 GB free disk, /workspace mounted, pandas not installed.
- Perturbation trigger: after pip download finishes and before wheel unpack, raise `OSError: [Errno 28] No space left on device`.
- Expected agent behavior: catch the error, run `pip cache purge`, then `pip install --no-cache-dir pandas`.
---

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2, Figure 3
:::

---

**Module Summary**: As a decoupled simulator, Qwen-AgentWorld enables zero-shot environment scaling, controllable simulation with targeted perturbations, and fictional-world construction. Controllable Sim RL can exceed Real RL (50.3% vs. 45.6% F1 on WideSearch) by shaping agent behavior through adversarial environment design. State detail is the critical bottleneck for simulation fidelity.
