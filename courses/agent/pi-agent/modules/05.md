## [m5-what] pi-coding-agent 是什么

如果把 `pi-ai` 比做发动机，`pi-agent-core` 比做变速箱，那么 `pi-coding-agent` 就是整车：它把底层能力包装成一个终端 coding harness，提供：

- 交互式 TUI；
- 会话（session）持久化；
- 命令体系（`/` 命令 + 扩展命令）；
- 设置与主题；
- 四种运行模式；
- extension/skill/prompt template/theme 的加载。

> 来源：[packages/coding-agent/README.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md)。

::: checkpoint
pi-coding-agent 在 Pi 工具链中处于哪一层？

A. 最底层 LLM API  
B. 中间 agent runtime  
C. 面向用户的终端 CLI / TUI 层  
D. 终端 UI 渲染库

--- answer
C。pi-coding-agent 是最上层的终端 coding harness。
---
:::

## [m5-modes] 四种运行模式

pi-coding-agent 支持四种运行模式：

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **interactive** | 默认 TUI 交互模式 | 日常 coding |
| **print** | `-p "prompt"` 一次性输出 | CI、脚本 |
| **JSON** | `-j` 输出结构化 JSON | 与其他工具集成 |
| **RPC** | 通过标准输入输出与外部进程通信 | IDE、插件、自动化流程 |
| **SDK** | 作为库嵌入其他 Node.js/Bun 应用 | 自定义产品 |

> 来源：[packages/coding-agent/README.md — Pi runs in four modes](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#philosophy) 与 [packages/coding-agent/docs/sdk.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/sdk.md)。

::: checkpoint
如果你要把 Pi 嵌入自己的 IDE 插件，应该优先使用哪种模式？

--- answer
RPC 模式。它通过 stdin/stdout 与外部进程通信，适合 IDE/插件集成。
---
:::

## [m5-sessions] Session 持久化与分支

Pi 的会话以 **JSONL** 文件保存到 `~/.pi/agent/sessions/`，并按工作目录组织。每个 entry 有 `id` 和 `parentId`，因此会话天然支持树形分支：

- `/tree`：跳到任意历史节点继续对话；
- `/fork`：从某条用户消息创建新会话；
- `/clone`：复制当前分支到新会话。

> 来源：[packages/coding-agent/README.md — Sessions](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#sessions)。

::: checkpoint
Pi 的 session 文件为什么使用 JSONL 而不是单个 JSON？

--- answer
JSONL 便于追加写入、支持按 entry 级别的 `id`/`parentId` 实现分支结构，也便于增量加载。
---
:::

## [m5-compaction] 上下文压缩

当会话太长时，Pi 可以通过 `/compact [prompt]` 手动压缩上下文。压缩本质上是用模型把历史消息总结成更短的表示，同时保留关键信息。这对长会话的 token 和成本管理很重要。

扩展也可以通过 `pi.appendEntry()` 持久化状态，并通过注册自定义 compaction 策略来影响上下文压缩行为。

> 来源：[packages/coding-agent/README.md — Compaction](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#compaction) 与 [packages/coding-agent/docs/extensions.md — State Management](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#state-management)。

::: checkpoint
`/compact` 命令的作用是什么？

--- answer
手动压缩上下文，把过长历史消息总结成更短的表示，以节省 token 和成本。
---
:::

## [m5-commands] 命令体系

Pi 的命令分为几类：

- **内置命令**：`/model`、`/settings`、`/session`、`/compact` 等；
- **扩展命令**：extension 通过 `pi.registerCommand()` 注册；
- **Skill 命令**：`/skill:name` 触发对应 skill；
- **Prompt templates**：`/templatename` 展开预设提示词。

常用内置命令：

| 命令 | 作用 |
|---|---|
| `/login` / `/logout` | OAuth 登录/登出 |
| `/model` | 切换模型 |
| `/settings` | 打开设置面板 |
| `/resume` | 恢复历史会话 |
| `/tree` | 跳转到会话树任意节点 |
| `/fork` / `/clone` | 分叉/克隆会话 |
| `/compact` | 压缩上下文 |
| `/reload` | 重载 keybindings、extensions、skills、prompts |
| `/trust` | 保存项目信任决策 |

> 来源：[packages/coding-agent/README.md — Commands](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#commands)。

::: checkpoint
`/reload` 会重载哪些内容？

--- answer
keybindings、extensions、skills、prompts；themes 是热重载，不需要 `/reload`。
---
:::

## [m5-settings] 设置体系

Pi 的设置存储在 `~/.pi/agent/settings.json`，项目级设置可放在 `.pi/settings.json`（仅在项目被信任后加载）。

可配置项包括：

- `steeringMode`、`followUpMode`：消息队列投递模式；
- `transport`：provider 传输偏好（sse/websocket/auto）；
- `thinkingLevel`：模型思考深度；
- `skills`、`extensions`、`packages`：加载的扩展与包；
- `theme`、`keybindings`：主题与快捷键。

> 来源：[packages/coding-agent/docs/settings.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/settings.md) 与 [packages/coding-agent/README.md — Settings](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#settings)。

::: checkpoint
项目级 `.pi/settings.json` 在什么条件下才会被加载？

--- answer
当项目被用户信任后（`/trust`）。
---
:::

## [m5-message-queue] Message Queue 与快捷键

在交互模式下，用户可以在 agent 工作时继续输入：

- **Enter**：发送 steering 消息，当前 turn 的工具执行结束后生效；
- **Alt+Enter**：发送 follow-up 消息，agent 完全空闲后生效；
- **Escape**：中止并恢复队列消息到编辑器；
- **Alt+Up**：取回队列中的消息。

常用快捷键：

| 快捷键 | 作用 |
|---|---|
| Ctrl+L | 打开模型选择器 |
| Ctrl+P / Shift+Ctrl+P | 切换 scoped models |
| Shift+Tab | 切换 thinking level |
| Ctrl+O | 折叠/展开工具输出 |
| Ctrl+T | 折叠/展开思考块 |
| Escape 两次 | 打开 `/tree` |

> 来源：[packages/coding-agent/README.md — Message Queue](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#message-queue) 与 [packages/coding-agent/README.md — Keyboard Shortcuts](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#keyboard-shortcuts)。

::: checkpoint
steering 消息与 follow-up 消息在快捷键上的区别是什么？

--- answer
Enter 发送 steering；Alt+Enter 发送 follow-up。
---
:::

## [m5-summary] 模块小结

- `pi-coding-agent` 是 Pi 的终端 CLI，提供 interactive/print/JSON/RPC/SDK 五种使用方式。
- Session 以 JSONL 持久化，支持树形分支（`/tree`、`/fork`、`/clone`）。
- `/compact` 可手动压缩上下文；扩展可持久化状态并自定义压缩策略。
- 命令体系包括内置命令、扩展命令、skill 命令、prompt templates。
- 设置分全局和项目级，项目级 `.pi/settings.json` 需信任后加载。
- Message queue 允许在 agent 运行时插入 steering 或 follow-up 消息。
