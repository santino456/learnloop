## [m4-deepspec] DeepSpec 仓库

::: concept
title: DeepSpec 是一个全栈训练与评估代码库
why: DSpark 不仅开源了权重，还开源了训练 draft 模型的完整流程。

DeepSpec 仓库（`deepseek-ai/DeepSpec`）包含：

- **三种投机解码算法实现**：DSpark、DFlash、Eagle3。
- **训练脚本**：数据准备、多 GPU 训练、评估。
- **配置驱动**：例如 `config/dspark/dspark_qwen3_4b.py` 定义模型与训练超参。
- **9 个离线 benchmark**：GSM8K、MATH、AIME25、HumanEval、MBPP、LiveCodeBench、MT-Bench、Alpaca、Arena-Hard-v2。

延伸阅读：
- [DeepSpec GitHub 仓库](https://github.com/deepseek-ai/DeepSpec)
- [HuggingFace DeepSeek-V4-Pro-DSpark 权重](https://huggingface.co/deepseek-ai/DeepSeek-V4-Pro-DSpark)
:::

## [m4-training] 训练流程

::: concept
title: 冻结目标模型，复用 embedding 与输出头
why: 不需要重新训练大模型，只需要训练一个外挂 draft 模块。

DSpark 的训练只更新 draft 模块参数：

- 目标模型（如 DeepSeek-V4）权重完全冻结。
- Draft backbone、sequential head、confidence head 从零或 Warmup 开始训练。
- Embedding 层和输出头复用目标模型，保证 draft 与 target 的 token 分布对齐。

这样做的好处是训练成本低，且不会破坏原模型的输出分布。
:::

::: concept
title: 数据准备与目标缓存
why: 训练 draft 需要目标模型对大量序列的分布信息。

训练前需要准备 **target cache**：用冻结的目标模型跑一遍训练数据，保存中间层特征与输出分布。这个缓存可能非常大——例如 Qwen3-4B 设置下接近 38 TB。缓存越大，训练数据覆盖越广，draft 泛化越好。

训练阶段默认使用单节点 8 GPU，可通过 `CUDA_VISIBLE_DEVICES` 调整可见设备数。
:::

::: evidence
claim: DSpark 的训练配置默认单节点 8 GPU，目标缓存可达数十 TB 量级。
source: [MarkTechPost: DeepSeek Releases DSpark](https://www.marktechpost.com/2026/06/27/deepseek-releases-dspark-a-speculative-decoding-framework-that-accelerates-deepseek-v4-per-user-generation-60-85-over-mtp-1/)
status: verified
basis: 报道明确提到默认 8 GPU 配置与约 38 TB target cache（Qwen3-4B 设置）。
:::

## [m4-deployment] 生产部署

::: concept
title: 已发布即插即用的 checkpoint
why: 大多数用户不需要自己训练，直接加载权重即可。

DeepSeek 在 HuggingFace 上发布了两个生产 checkpoint：

- `deepseek-ai/DeepSeek-V4-Pro-DSpark`
- `deepseek-ai/DeepSeek-V4-Flash-DSpark`

它们不是新训练的目标模型，而是原 V4 checkpoint 加上训练好的 draft 模块。推理框架可以直接加载并启用投机解码。
:::

::: concept
title: vLLM / SGLang 服务方式
why: 主流开源推理引擎已支持或正在集成 DSpark。

示例启动命令：

```bash
# vLLM
vllm serve "deepseek-ai/DeepSeek-V4-Pro-DSpark"

# SGLang
python3 -m sglang.launch_server \
  --model-path "deepseek-ai/DeepSeek-V4-Pro-DSpark"
```

实际部署时还需要配置 speculative decoding 参数，如 draft 长度、验证 batch size、置信度阈值等。

延伸阅读：
- [vLLM Issue: Support DSpark for DeepSeek V4](https://github.com/vllm-project/vllm/issues/46910)
- [vLLM Issue: DSpark performance](https://github.com/vllm-project/vllm/issues/46931)
:::

::: exercise
把训练/部署阶段与主要成本连起来。

- 准备 target cache
- 训练 draft 模块
- 在线服务加载 DSpark checkpoint

成本：

1. 大量存储空间
2. 8 GPU 训练时间
3. 几乎零额外训练成本

- 准备 target cache → 1
- 训练 draft 模块 → 2
- 在线服务加载 DSpark checkpoint → 3

:::

## [m4-summary] 本章小结

- DeepSpec 提供 DSpark / DFlash / Eagle3 的全栈训练与评估代码。
- DSpark 冻结目标模型，只训练 draft 模块，训练成本低。
- 目标缓存可能很大，但生产 checkpoint 可以直接加载使用。
- vLLM、SGLang 等引擎正在集成，部署门槛逐步降低。
