## [m4-bench] AgentWorldBench: A Comprehensive LWM Benchmark

AgentWorldBench is constructed to evaluate language world models across seven agent interaction domains. It contains 2,170 turn-level evaluation samples built from 5 frontier models on 9 established benchmarks.

::: figure
src: assets/qwen-agentworld-fig-6.png
alt: Figure 6 overview of AgentWorldBench composition showing domain distribution, source benchmarks, evaluation dimensions, and summary statistics.
caption: Figure 6: Overview of AgentWorldBench composition. Left: Domain distribution across seven domains, source benchmarks mapped to each domain, and the five evaluation dimensions (Format, Factuality, Consistency, Realism, Quality). Right: Summary statistics, per-domain average context length and trajectory depth. All ground-truth observations are obtained from real environment execution.
:::

::: concept
title: Construction principles

**Construction principles**:
1. **Widely-used queries**: All task queries are drawn from established high-quality agentic benchmarks.
2. **Frontier-agent trajectories**: All trajectories are generated by frontier-model agents.
3. **Real observations**: Every trajectory is paired with ground-truth observations from real environment execution.
4. **Out-of-distribution**: Training data and benchmark queries are partitioned at the data-source level.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.1
:::

## [m4-protocol] Evaluation Protocol

AgentWorldBench evaluates simulation quality through an open-ended rubric: an LLM judge scores each predicted observation on five dimensions.

::: concept
title: Five evaluation dimensions

**Five evaluation dimensions** (each 1–5, normalized to 0–100):
- **Format**: Structural conventions of the domain (JSON schema compliance, shell prompt patterns).
- **Factuality**: Correctness of stated facts (file contents, search results, tool return values).
- **Consistency**: Internal coherence and coherence with prior turns.
- **Realism**: Behavioral characteristics of the real environment (response patterns, style conventions, value plausibility).
- **Quality**: Completeness and conciseness relative to the ground truth.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2
:::

## [m4-judging] Reference-Grounded LLM Judging

::: concept
title: Reference-grounded judging

**Reference-grounded judging**: The judge receives the ground-truth environment observation alongside the predicted observation and scores each dimension by comparing the two. This converts evaluation from an open-ended quality judgment into a factual comparison, substantially narrowing the space for judge hallucination or misjudgment.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2
:::

::: concept
title: Differentiated matching criteria

**Differentiated matching criteria**: Not all content requires exact matching. Content is classified into three types before evaluation:
- **Deterministic content** (echo output, file reads, computation results): must match exactly.
- **Pre-existing environment content** (preinstalled software versions, file contents not created by the trajectory): format and plausibility verification only.
- **Runtime metadata** (timestamps, PIDs, memory addresses): format and range verification only.

This three-way split lets the judge reward correct structural and semantic behavior without penalizing irreproducible details.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2
:::

## [m4-calibration] Judge Calibration via Double-Blind Turing Tests

::: concept
title: Double-blind Turing test calibration

**Double-blind Turing test calibration**: Given the interaction history and current action, the judge receives two candidate observations in randomized order—one from the real environment and one from the world model—and must identify which came from the real environment. A judge with high Turing-test accuracy must attend to fine-grained differences between faithful and plausible-but-wrong simulation, the same discriminative ability required for accurate rubric scoring.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2
:::

**Cross-judge consistency**: Three frontier models (Gemini 3 Flash, Claude Sonnet 4.5, GPT-5.2) were compared as judge candidates. Despite systematic absolute differences (Gemini most lenient, GPT-5.2 most stringent), the pairwise Spearman rank correlations are ρ = 0.92–0.99 (all p < 10^-5), indicating that the three judges agree on which models produce higher-fidelity predictions despite assigning different absolute scores. GPT-5.2 was selected as the final judge for its highest average Turing-test accuracy.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2

**Key results**:
- Qwen-AgentWorld-397B-A17B achieves the highest overall average (58.71), surpassing GPT-5.4 (58.25) and Claude Opus 4.6 (57.80).
- On text-based domains, Qwen-AgentWorld leads with 58.07 average (vs. GPT-5.4 at 56.84).
- On GUI domains, Claude Opus 4.8 leads (60.93), with Qwen-AgentWorld ranking fifth (59.69). The gap is attributed to multimodal pre-training advantages, though this interpretation was not experimentally isolated.
- Search is the weakest domain for all models: best score 37.82 vs. 68.49 on SWE.
- World-model training improves Qwen3.5-35B-A3B from 47.73 to 56.39 (+8.66 points); at 397B scale from 54.74 to 58.71 (+3.97 points).

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §5.2

## [m4-exercise] Exercise: Design a Judge Prompt

**Task**: Design a domain-aware rubric judge prompt for the MCP domain following the five-dimensional framework.

**Requirements**:
1. Define each of the five dimensions (Format, Factuality, Consistency, Realism, Quality) in MCP-specific terms.
2. Include a content-type classification table with at least three content types and their verification standards.
3. Specify differentiated matching criteria for deterministic vs. non-deterministic content.

**Verification criteria**: The Factuality dimension should emphasize JSON schema compliance and tool execution correctness. The Consistency dimension should require cross-turn resource reference integrity (e.g., IDs created in earlier turns must match in later turns).

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §4.2, Appendix D.2

---

**Module Summary**: AgentWorldBench evaluates world models via reference-grounded, five-dimensional rubric judging with differentiated matching criteria. Cross-judge consistency is high (ρ = 0.92–0.99), validating the reliability of the benchmark. Qwen-AgentWorld leads on text-based domains; GUI gaps may reflect multimodal pre-training advantages.
