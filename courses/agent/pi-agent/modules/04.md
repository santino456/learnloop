## [m4-why] 终端 UI 的痛点

在终端里做交互式 UI，最常见的两个问题是：

1. **闪烁**：每次全屏重绘时，屏幕会短暂出现空白或残影。
2. **性能**：全屏重绘在复杂界面下既慢又费资源。

`pi-tui` 的定位是 **minimal terminal UI framework with differential rendering and synchronized output for flicker-free interactive CLI applications**。

> 来源：[packages/tui/README.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md)。

::: checkpoint
pi-tui 主要解决哪两个终端 UI 痛点？

--- answer
闪烁和性能。它通过差分渲染和同步输出（CSI 2026）解决。
---
:::

## [m4-differential] 差分渲染与同步输出

pi-tui 采用三种策略的差分渲染系统，只更新变化的部分，而不是每次都清屏重绘。同时它使用 **CSI 2026** 转义序列实现原子屏幕更新，避免半帧画面被用户看到。

差分渲染带来的直接好处：

- 工具调用日志滚动时更平滑；
- 模型流式输出时不会闪屏；
- 复杂 overlay（如设置面板、模型选择器）弹出时保持底层内容稳定。

> 来源：[packages/tui/README.md — Features](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md#features)。

::: checkpoint
CSI 2026 在 pi-tui 中的作用是什么？

--- answer
CSI 2026 用于同步输出，让终端把一帧更新作为原子操作处理，避免闪烁。
---
:::

## [m4-component] 组件模型

pi-tui 的组件接口非常轻量：

```typescript
interface Component {
  render(width: number): string[];
  handleInput?(data: string): void;
  invalidate?(): void;
}
```

- `render(width)` 返回一个字符串数组，每行一个字符串，且每行不能超过 `width`。
- `handleInput` 接收原始终端输入（可能包含 ANSI 转义序列）。
- `invalidate` 用于清空缓存的渲染状态。

> 来源：[packages/tui/README.md — Component Interface](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md#component-interface)。

::: checkpoint
一个 pi-tui 组件最少需要实现哪个方法？

--- answer
`render(width)`。
---
:::

## [m4-focus] Focusable 与硬件光标

需要显示文本光标的组件可以实现 `Focusable` 接口。当组件获得焦点时：

1. TUI 设置 `focused = true`；
2. 扫描渲染输出中的 `CURSOR_MARKER`（零宽 APC 转义序列）；
3. 把硬件光标定位到该位置；
4. 默认隐藏硬件光标，但可开启以兼容 IME。

`Editor` 和 `Input` 组件已经实现了这个接口。

> 来源：[packages/tui/README.md — Focusable Interface](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md#focusable-interface-ime-support)。

::: checkpoint
`CURSOR_MARKER` 在 pi-tui 中的作用是什么？

--- answer
它是一个零宽 APC 转义序列，用于标记硬件光标应该放置的位置。
---
:::

## [m4-overlay] Overlay 系统

pi-tui 支持在底层内容之上渲染 overlay，不替换底层内容。典型用途：

- `/model` 模型选择器；
- `/settings` 设置面板；
- 扩展自定义的对话框、菜单。

Overlay 支持多种定位方式：anchor、百分比、绝对坐标、margin 限制，还可以根据终端尺寸动态决定是否显示。

> 来源：[packages/tui/README.md — Overlays](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md#overlays)。

::: checkpoint
pi-tui 的 Overlay 为什么不会替换底层内容？

--- answer
Overlay 是渲染在现有内容之上的层，适合弹窗、菜单等临时 UI，底层内容保持不变。
---
:::

## [m4-pi-usage] Pi 如何使用 pi-tui

`pi-coding-agent` 的交互界面完全基于 `pi-tui`：

- 顶部 header、消息列表、编辑器、底部状态栏都是组件；
- 流式文本输出通过 `Text` 组件增量更新；
- 工具调用结果通过自定义渲染组件展示；
- `/model`、`/settings`、扩展自定义 UI 都使用 overlay；
- 快捷键、思考块折叠、工具输出折叠都通过 `handleInput` 处理。

理解 `pi-tui` 有助于你在写 extension 时自定义 UI（例如用 `ctx.ui.custom()` 实现复杂交互）。

> 来源：[packages/coding-agent/README.md — Interactive Mode](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/README.md#interactive-mode) 与 [packages/coding-agent/docs/extensions.md — Custom UI](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#custom-ui)。

::: checkpoint
在 extension 中，哪个 API 允许你使用 pi-tui 构建复杂自定义 UI？

--- answer
`ctx.ui.custom()`。
---
:::

## [m4-summary] 模块小结

- `pi-tui` 是 Pi 的终端 UI 库，核心特性是差分渲染和 CSI 2026 同步输出。
- 组件接口极简：`render(width)`、`handleInput`、`invalidate`。
- `Focusable` 接口支持硬件光标定位和 IME；`Editor`/`Input` 已内置实现。
- Overlay 系统支持弹窗、菜单等临时 UI，不影响底层内容。
- `pi-coding-agent` 的所有交互界面都建立在 `pi-tui` 之上，extension 可以通过 `ctx.ui.custom()` 复用它。

> 来源：本地 m3《pi-agent-core：状态化 Agent Loop》；外部 [packages/tui/README.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/tui/README.md)。
