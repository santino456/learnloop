## [m2-what] pi-ai 解决什么问题

如果你想写一个支持多家 LLM 的 agent，最直接的麻烦是：每家 API 的认证方式、消息格式、工具调用协议、流式事件都不一样。`pi-ai` 把这些差异封装成一个统一的接口：

- 统一的 `Model` 与 `Models` 集合；
- 统一的 `Context`（system prompt + messages + tools）；
- 统一的流式事件（`text_delta`、`toolcall_start/end`、`thinking_delta`、`done`、`error` 等）；
- 统一的工具参数校验（基于 TypeBox）；
- 统一的认证解析（环境变量、OAuth、自定义 store）。

> 来源：[packages/ai/README.md — @earendil-works/pi-ai](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md)。

::: checkpoint
pi-ai 封装的主要差异不包括以下哪一项？

A. 多厂商认证方式  
B. 流式事件格式  
C. 代码版本管理  
D. 工具调用协议

--- answer
C。pi-ai 不处理代码版本管理，它封装的是 LLM API 差异。
---
:::

## [m2-models] Models 与 Provider 工厂

`pi-ai` 的核心入口是 `Models` 集合。你可以注册所有内置 provider，也可以只注册需要的 provider 以减小打包体积。

```typescript
import { builtinModels } from '@earendil-works/pi-ai/providers/all';
import { openaiModels } from '@earendil-works/pi-ai/providers/openai';

const all = builtinModels();
const onlyOpenAI = openaiModels();
```

从集合中读取模型后，得到一个 `Model` 对象，后续所有调用都基于这个对象：

```typescript
const model = all.getModel('openai', 'gpt-4o-mini')!;
```

> 来源：[packages/ai/README.md — Provider Factories](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#provider-factories)。

内置 provider 包括 OpenAI、Anthropic、Google、Azure OpenAI、DeepSeek、Groq、Cerebras、Mistral、Together、OpenRouter、xAI、Cloudflare、Amazon Bedrock、Hugging Face、Kimi、MiniMax、Xiaomi MiMo 等。具体列表见仓库 README。

> 来源：[packages/ai/README.md — Supported Providers](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#supported-providers)。

::: checkpoint
`builtinModels()` 与 `openaiModels()` 的主要区别是什么？

--- answer
`builtinModels()` 注册所有内置 provider；`openaiModels()` 只注册 OpenAI，适合需要减小打包体积的场景。
---
:::

## [m2-context] Context：可序列化的对话上下文

`Context` 是 `pi-ai` 的核心抽象：

```typescript
type Context = {
  systemPrompt: string;
  messages: Message[];
  tools: Tool[];
};
```

它的一个重要特性是 **可序列化、可迁移**。你可以把 context 存下来、传给另一个模型，甚至在不同 provider 之间切换继续对话。

```typescript
const context: Context = {
  systemPrompt: 'You are a helpful assistant.',
  messages: [{ role: 'user', content: 'What time is it?', timestamp: Date.now() }],
  tools: [{
    name: 'get_time',
    description: 'Get the current time',
    parameters: Type.Object({
      timezone: Type.Optional(Type.String({ description: 'Optional timezone' }))
    })
  }]
};
```

> 来源：[packages/ai/README.md — Quick Start](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#quick-start)。

::: checkpoint
Context 最重要的工程特性是什么？

A. 它只能用于单个 provider  
B. 它不可序列化  
C. 它可序列化并在模型之间迁移  
D. 它不包含 tools

--- answer
C。Context 设计为可序列化，支持切换模型、持久化、跨 provider 迁移。
---
:::

## [m2-tools] Tool 定义与参数校验

pi-ai 使用 TypeBox 定义工具参数，提供类型安全并在运行时校验：

```typescript
import { Type } from '@earendil-works/pi-ai';

const tools = [{
  name: 'get_time',
  description: 'Get the current time',
  parameters: Type.Object({
    timezone: Type.Optional(Type.String({ description: 'Optional timezone' }))
  })
}];
```

流式返回的工具参数是部分 JSON，`pi-ai` 提供 `validateToolArguments` 在完整参数可用时进行校验。

> 来源：[packages/ai/README.md — Defining Tools](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#defining-tools) 与 [packages/ai/README.md — Validating Tool Arguments](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#validating-tool-arguments)。

::: checkpoint
pi-ai 用什么库做工具参数的类型定义和运行时校验？

--- answer
TypeBox。pi-ai 甚至从 `@earendil-works/pi-ai` 重新导出了 `Type`、`Static`、`TSchema`。
---
:::

## [m2-streaming] 统一流式事件

`Models.stream(model, context)` 返回一个异步事件流。关键事件类型包括：

| 事件 | 含义 |
|---|---|
| `start` | 流开始 |
| `text_start` / `text_delta` / `text_end` | 文本生成 |
| `thinking_start` / `thinking_delta` / `thinking_end` | 模型思考/推理内容 |
| `toolcall_start` / `toolcall_delta` / `toolcall_end` | 工具调用 |
| `done` | 正常结束 |
| `error` | 出错 |

```typescript
const s = models.stream(model, context);
for await (const event of s) {
  if (event.type === 'text_delta') process.stdout.write(event.delta);
}
const finalMessage = await s.result();
context.messages.push(finalMessage);
```

> 来源：[packages/ai/README.md — Quick Start](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#quick-start)。

::: checkpoint
流式工具调用参数到达时，对应的事件类型是什么？

--- answer
`toolcall_delta`。完整工具调用在 `toolcall_end` 时可用。
---
:::

## [m2-auth] 认证解析

pi-ai 的认证不是写死的环境变量读取，而是按优先级解析：

1. 代码中显式传入的 API key；
2. 凭证 store（`CredentialStore`）；
3. 环境变量（如 `OPENAI_API_KEY`）；
4. OAuth 登录获取的 token。

不同 provider 支持不同的认证方式。详情见 `packages/ai/src/auth/resolve.ts`。

> 来源：[packages/ai/README.md — How Auth Resolves](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#how-auth-resolves) 与 [packages/ai/src/auth/resolve.ts](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/src/auth/resolve.ts)。

::: checkpoint
pi-ai 的认证解析优先级中，最高优先级是什么？

--- answer
代码中显式传入的 API key。
---
:::

## [m2-custom-provider] 自定义 Provider

如果内置 provider 不满足需求，可以用 `createProvider()` 构造自己的 provider，或者直接调用 API 实现。pi-ai 把 provider 拆成：

- **Provider factories**：生成 `Model` 集合；
- **API implementations**：实际发起 HTTP 请求；
- **Compat 层**：兼容旧 API。

自定义 provider 需要实现 `Provider` 接口，返回 `Model` 对象，并指定对应的 stream 函数。

> 来源：[packages/ai/README.md — Custom Providers](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md#custom-providers) 与 [packages/ai/src/index.ts](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/src/index.ts)。

::: checkpoint
自定义 provider 时，最低限度需要实现什么？

--- answer
实现 `Provider` 接口并返回 `Model` 对象，同时提供对应的 stream 函数。
---
:::

## [m2-summary] 模块小结

- `pi-ai` 是 Pi 的底层 LLM 抽象层，统一了 model、context、tool、streaming、auth。
- `Models` 集合负责 provider 注册和模型查找；`Context` 是可序列化的对话上下文。
- 工具参数用 TypeBox 定义，流式事件覆盖文本、思考、工具调用、错误等状态。
- 认证按显式 key > credential store > 环境变量 > OAuth 的优先级解析。
- 自定义 provider 通过 `createProvider()` 或 API implementations 实现。

> 来源：本地 m1《Pi 是什么，为什么值得二次开发》；外部 [packages/ai/README.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/ai/README.md)。
