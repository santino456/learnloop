## [m3-hc-background] 从 Hyper-Connections 说起

Hyper-Connections（HC）把相邻 Transformer 块之间的残差连接解耦成三个映射：

- 输入映射 A
- 残差映射 B
- 输出映射 C

它让“残差宽度”可以独立于隐藏层维度扩展，为模型提供了一个额外的缩放维度。但问题是：当堆叠很多层时，HC 容易出现数值不稳定。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 2.2 节。

## [m3-manifold] mHC 的核心思想

Manifold-Constrained Hyper-Connections（mHC）的关键是把残差映射矩阵 **B_l 约束到双随机矩阵流形**（Birkhoff polytope）：

- 每行、每列之和为 1
- 所有元素非负

这样的矩阵 spectral norm 不超过 1，保证残差变换是**非扩张的**（non-expansive），从而提升深层网络的前向传播和反向传播稳定性。

::: checkpoint
为什么把 B_l 约束为双随机矩阵能提升稳定性？

--- answer
双随机矩阵的谱范数 ≤ 1，因此残差映射不会放大信号；深层堆叠时信号不会爆炸，前向和反向都更稳定。
---
:::

## [m3-dynamic] 动态参数化

A、B、C 三个映射的参数是动态生成的，但拆成了两部分：

- **动态分量**：依赖当前输入
- **静态分量**：与输入无关

具体步骤：

1. 把输入 X_l 展平并做 RMSNorm，得到 ˆX_l。
2. 用 ˆX_l 分别与可学习的 W_pre、W_res、W_post 相乘，得到原始参数 ˜A、˜B、˜C。
3. 加上可学习的静态偏置 S_pre、S_res、S_post。

输入映射 A 和输出映射 C 再通过 Sigmoid 函数保证非负有界：

- A = σ(˜A)
- C = 2σ(˜C)

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 2.2 节。

## [m3-sinkhorn] Sinkhorn-Knopp 投影

对于残差映射 ˜B，需要把它投影到双随机矩阵流形。做法是先取指数保证正性，然后反复进行行归一化和列归一化：

```
M^(0) = exp(˜B)
M^(t) = T_r(T_c(M^(t-1)))
```

这就是 Sinkhorn-Knopp 算法，能把一个正矩阵迭代收敛到双随机矩阵。

::: flow
原始参数 ˜B
  -> 取指数得到正矩阵 M^(0)
  -> 行归一化 T_c
  -> 列归一化 T_r
  -> 迭代收敛到双随机矩阵 B_l
:::

## [m3-summary] 模块小结

- mHC 在 HC 基础上增加了流形约束，解决深层堆叠的数值不稳定问题。
- 残差映射 B_l 被约束为双随机矩阵，保证非扩张性。
- A 和 C 通过 Sigmoid 约束；B 通过 Sinkhorn-Knopp 投影实现。
- 动态参数化让映射既能适应输入，又保留稳定的静态偏置。
