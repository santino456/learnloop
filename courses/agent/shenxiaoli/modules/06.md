## [m6-tools] ToolRegistry：7 个工具的注册与分发

`ToolRegistry` 是深小理的工具中枢——把 Python 函数注册为 Agent 可调用的 tool definition。

### 工具清单

| 工具 | 参数 | 返回 | 隐私敏感 |
|------|------|------|---------|
| `read_file` | `path` | 文件内容 | 否 |
| `search` | `query` | BM25 搜索结果（top-5） | 否 |
| `list_files` | `dir` | 目录列表 | 否 |
| `execute` | `script_name, args` | 脚本输出 | 否 |
| `ask_user` | `question, component, options, multi, max_score, context` | 暂停等待用户 | 否（但会中断 Agent 循环） |
| `record_observation` | `content, category` | 确认消息 | **是** |
| `update_user_memory` | `content` | 确认消息 | **是** |

工具注册时自动生成 OpenAI 兼容的 `tool_definition`：

```python
{
    "type": "function",
    "function": {
        "name": "search",
        "description": "搜索知识库",
        "parameters": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "搜索关键词"}
            },
            "required": ["query"]
        }
    }
}
```

## [m6-bm25] BM25 搜索：一个轻量中文检索引擎

`search` 工具背后是一套手写的 BM25 搜索引擎——**没有 Elasticsearch，没有向量数据库**。

### 架构

```
知识库 Markdown 文件
  ↓
按 ## / ### 标题切分为 chunks（带 breadcrumb）
  ↓
中文 2-gram + 英文单词 分词
  ↓
BM25 打分（k1=1.5, b=0.75）
  ↓
排序 + 去重 + 过滤 → top-5 结果
```

### 打分策略

| 因素 | 方式 | 权重 |
|------|------|------|
| BM25 | TF-IDF 变体 | 基准 |
| 标题命中 | chunk 标题包含查询词 | +5 |
| 文件名命中 | 来源文件名包含查询词 | +3 |
| 短内容惩罚 | chunk 内容 < 50 字符 | ×0.7 |
| 去重 | 同一 breadcrumb 最多 2 条 | - |
| 分数门槛 | 低于最高分 20% 的结果丢弃 | - |

### 为什么不用向量检索

- 知识库规模：几百篇文章，BM25 精度足够
- 没有 GPU 依赖：BM25 纯 CPU 计算
- 部署零成本：一段 Python 代码，不需要单独的服务
- 中文好处理：2-gram 分词对任意中文文本都有效，不需要分词词典

::: checkpoint
BM25 和向量检索的核心区别是什么？什么场景下 BM25 不够用？

--- answer
BM25 依赖词频匹配——用户问"图书馆"就找含"图书馆"的文档。向量检索依赖语义相似——用户问"学习的地方"，即使没出现"图书馆"这个词，也能通过向量相似度找到相关文档。

BM25 不够用的场景：1) 同义词/近义词查询（"借书" vs "图书馆"）；2) 跨语言查询；3) 大规模文档（成千上万篇）时 BM25 稀疏性导致召回不足。深小理的知识库只有几百篇中文文章，BM25 足够。
---
:::

## [m6-skills] Skill 系统：场景化能力扩展

Skill 是比 Tool 更高层的抽象——一个 Skill 包含多个工具、脚本和参考资料。

### 加载机制

```
workspace/skills/
├── admission-consultant/   ← 招生咨询
│   ├── SKILL.md           ← 被 PersonalityLoader 加载到 system prompt
│   └── (可选: scripts/, references/, assets/)
├── campus-info/            ← 校园资讯
├── campus-tour/            ← 校园导览
├── poetry/                 ← 写诗
└── story/                  ← 讲故事
```

`SkillRegistry` 在启动时自动扫描 `workspace/skills/` 目录。每个包含 `SKILL.md` 的目录被识别为一个 Skill。解析 YAML 风格 frontmatter（手写解析器，不依赖 PyYAML）提取 `name` 和 `description`。

生成的 manifest 注入到 system prompt 中：

```
可用技能：
- admission-consultant: 招生咨询、录取分数、专业选择...
- campus-info: 校园活动、学术讲座、新闻动态...
...
重要：使用任何技能之前，必须先用 read_file 加载其 SKILL.md。
```

### Skill 的结构模式

所有 Skill 的 `SKILL.md` 遵循相似结构：

```
---
name: skill-name
description: when to trigger
---

# Skill Title
Persona and behavioral instructions...

## Core capabilities / workflow
## Red lines (hard bans)
```

- **Persona**：这个 Skill 激活时 Agent 的身份和行为
- **Workflow**：标准操作流程（先查什么、再做什么、怎么组织回答）
- **Red lines**：绝对不能做的行为（如招生 skill 不能承诺录取）

::: checkpoint
阅读 `workspace/skills/campus-tour/SKILL.md`（用 read_file 工具），然后回答：这个 Skill 的 Persona 是什么？Red lines 有哪些？

--- answer
Persona: 校园向导，用图片 + 互动引导展示校园，每条消息最多 2-3 张图片。
Red lines: 1) 不能编造图片路径；2) 不能在没有图片的情况下说"请看这张图"；3) 不能在未知位置的情况下猜测图片存在。
---
:::

## [m6-personality] PersonalityLoader：分层人格

`PersonalityLoader` 负责加载和缓存工作区文件。每个文件有字符上限，超出则截断：

| 文件 | 上限 | 作用 |
|------|------|------|
| `SOUL.md` | 1000 | 核心人格定义 |
| `SUAT.md` | 6000 | 校园事实数据 |
| `MEMORY.md` | 3500 | Agent 经验记忆 |
| `MESSAGE_FORMAT.md` | 1500 | 输出格式规范 |
| `interaction_policy.md` | (无限制) | 交互原则 |
| `tool_guidelines.md` | (无限制) | 工具使用规范 |

截断策略：在最后一个完整的行处截断，追加 `...(已截断)`——保证不切在句子中间。

缓存策略：按文件 mtime 缓存。文件修改后下次加载自动刷新。

## [m6-filesecurity] FileManager：文件访问安全

`FileManager` 是所有文件操作的中转站。安全策略：

- **路径解析**：相对路径 → 绝对路径（基于 workspace 根目录）
- **逃逸防护**：`resolved.relative_to(workspace_root)` 如果抛出 `ValueError`，说明路径试图逃逸
- **执行限制**：`execute` 只允许 `skills/` 目录下的 `.py` 文件
- **超时**：脚本执行 30 秒超时，`asyncio.create_subprocess_exec` + `asyncio.wait_for`

## [m6-summary] 模块小结

- ToolRegistry：7 个工具，注册时自动生成 OpenAI tool definition。
- BM25 搜索：自研中文检索引擎——没有 ES，没有向量数据库，几百篇文章场景精度够用。
- Skill：场景化能力扩展（招生、校园、写诗、讲故事），启动时自动扫描注册。
- PersonalityLoader：分层人格文件，mtime 缓存 + 按行截断。
- FileManager：路径安全守卫，禁止逃逸 workspace。
