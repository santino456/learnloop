## [m7-self-correction] Deliberative Self-Correction

::: concept
title: "Wait!" self-correction

**"Wait!" self-correction**: The model uses "Wait!" as an explicit cognitive interrupt to re-examine an intermediate prediction and revise before committing. Across 129 turns, there are 1,347 such interrupts (10.4 per turn on average; peak: 56 in a single SWE turn). Terminal and MCP exhibit the highest rates (16.9 and 12.7 per turn), reflecting deeper state-tracking demands.

The self-corrections decompose into three functional subtypes:
- **Factual**: catching an incorrect API response format.
- **Epistemological**: recognizing the limit of in-context computation (e.g., the np.random.seed(42) example).
- **Perspective-taking**: modeling the evaluator’s intent or the agent’s knowledge state.

This behavior converts environment prediction from single-pass generation into constrained satisfiability search.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-leakage] Information Leakage Prevention

::: concept
title: Information leakage prevention

**Information leakage prevention**: In the Search domain, the model holds a reference answer that the agent is trying to find. When the agent’s query is unrelated to this answer, the model explicitly prevents leakage: it identifies the topic mismatch and ensures that generated snippets do not accidentally reveal the target information. This is the world-model equivalent of theory of mind: the model distinguishes what the agent knows from what the environment should reveal.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-epistemic] Epistemic Boundary Awareness

::: concept
title: Epistemic boundary awareness

**Epistemic boundary awareness**: The model recognizes computational limits and falls back to format-only output rather than fabricating unknowable values. For example, when asked to predict values from np.random.randn(500) beyond the seed-known indices, the model recalls that the first values from seed(42) are well-known, but the 500th value is not, and therefore outputs format only rather than fabricating exact numbers.

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-timeline] RL Training Dynamics

::: timeline
RL Training Dynamics (Qwen-AgentWorld-35B-A3B, 440 steps)

~0 steps: Initial state — all five dimensions start from baseline SFT scores.
~90 steps: Format converges — reaches 90% of total improvement. Formatting conventions (JSON structure, terminal prompt strings) are surface patterns that RL reinforces quickly.
~250 steps: Consistency converges — requires deeper cross-turn state tracking capability that develops gradually.
440 steps: Final state — Factuality shows largest relative improvement (+11.3%) but remains lowest-scoring dimension. Quality improves least (+0.11).

## [m7-mapping] Mapping Reasoning Patterns to Downstream Capabilities

::: concept
title: Reasoning-to-capability mapping

**Reasoning-to-capability mapping**: The three reasoning patterns analyzed in this module directly enable downstream agent capabilities:
- **Deliberative self-correction** → Agent avoids committing to infeasible actions by internally simulating and revising predictions.
- **Information leakage prevention** → Agent maintains clean separation between its own knowledge and what it should derive from environment interaction.
- **Epistemic boundary awareness** → Agent recognizes when it lacks sufficient information and falls back to safe actions (e.g., requesting clarification or additional context).

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7
:::
