# Course Architecture

## Learner Goal

Help a technical learner understand ACP as a local client-to-agent integration boundary, then judge when it is useful.

## Module Plan

| Module | Learning job | Content forms | Why |
|--------|--------------|---------------|-----|
| m1 | Establish the core ACP idea | tutorial | The learner needs a mental model first. |
| m2 | Look up protocol roles and message shape | reference | The module contains dense reusable protocol facts and tables. |
| m3-m4, m6 | Practice message handling and implementation boundaries | practice | The learner needs to check operational understanding. |
| m5, m7, m8 | Extract integration judgment | perspective | These modules train protocol choice, boundary taste, and AI-era judgment. |

## Content Form Decisions

- Reference is used only for m2 because it contains dense lookup facts, tables, and lifecycle details.
- Perspective modules must state their basis because they express judgment rather than raw facts.
- Practice modules should train real decisions or implementation moves, not trivia.
