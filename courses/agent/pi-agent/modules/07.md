## [m7-what] Skill 是什么

Skill 是 **自包含的能力包**，包含：

- 一段描述（何时使用该 skill）；
- 使用说明；
- 可选的 helper 脚本、参考文档、资源文件。

与 Extension 不同，Skill 不直接执行代码或拦截事件，而是**通过提示词指导模型**。它适合封装：

- 特定领域的工作流程；
- 常用命令组合；
- 项目特定的约定或检查清单。

> 来源：[packages/coding-agent/docs/skills.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md)。

::: checkpoint
Skill 与 Extension 的最大区别是什么？

A. Skill 可以写 TypeScript 代码  
B. Skill 通过提示词指导模型，Extension 直接改运行时行为  
C. Skill 只能全局加载  
D. Skill 不能包含脚本

--- answer
B。Skill 主要是“给模型看的说明书”，Extension 是“直接改 Pi 运行时”。
---
:::

## [m7-standard] Agent Skills 标准

Pi 实现了 [Agent Skills 标准](https://agentskills.io/specification)，但有一个宽松点：Pi 允许 skill 名称与父目录不一致，因为标准要求名称必须匹配目录，这对多个 agent harness 共享 skill 目录不友好。

> 来源：[packages/coding-agent/docs/skills.md — How Skills Work](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md#how-skills-work)。

::: checkpoint
Pi 对 Agent Skills 标准做了哪项宽松处理？

--- answer
允许 skill 名称与父目录不一致。
---
:::

## [m7-loading] Skill 加载机制

Pi 加载 skill 的过程是渐进式的：

1. 启动时扫描 skill 位置，提取名称和描述；
2. 把可用 skills 以 XML 格式写入系统提示词；
3. 当任务匹配时，模型使用 `read` 加载完整 `SKILL.md`；
4. 模型按 SKILL.md 中的指示行动。

这意味着：**只有描述常驻上下文，完整指令按需加载**。既避免上下文膨胀，又能让模型在需要时拿到详细指南。

> 来源：[packages/coding-agent/docs/skills.md — How Skills Work](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md#how-skills-work)。

::: checkpoint
为什么 Pi 不把完整 SKILL.md 内容一直放在系统提示词里？

--- answer
为了减少上下文膨胀。只有 skill 描述常驻，完整内容在模型判断需要时才读取。
---
:::

## [m7-structure] SKILL.md 结构

一个 skill 目录至少包含 `SKILL.md`：

```text
my-skill/
├── SKILL.md              # 必需：frontmatter + 指令
├── scripts/              # 可选：helper 脚本
│   └── process.sh
├── references/           # 可选：详细参考文档
│   └── api-reference.md
└── assets/               # 可选：资源文件
    └── template.json
```

`SKILL.md` 必须包含 frontmatter：

```markdown
---
name: my-skill
description: What this skill does and when to use it. Be specific.
---

# My Skill

## Setup
Run once before first use:
```bash
cd /path/to/skill && npm install
```

## Usage
```bash
./scripts/process.sh <input>
```
```

> 来源：[packages/coding-agent/docs/skills.md — Skill Structure](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md#skill-structure)。

::: checkpoint
SKILL.md 中哪两个 frontmatter 字段是必需的？

--- answer
`name` 和 `description`。
---
:::

## [m7-locations] Skill 加载位置

| 位置 | 作用域 |
|---|---|
| `~/.pi/agent/skills/` 或 `~/.agents/skills/` | 全局 |
| `.pi/skills/` 或 `.agents/skills/`（cwd 及祖先目录） | 项目级（需信任） |
| `skills/` 目录或 `package.json` 的 `pi.skills` | Pi Packages 内 |
| `settings.json` 的 `skills` 数组 | 显式配置 |
| CLI `--skill <path>` | 临时加载 |

注意不同目录下 discovery 规则略有差异：

- 在 `~/.pi/agent/skills/` 和 `.pi/skills/` 中，根目录下的 `.md` 文件会被视为独立 skill；
- 所有 skill 位置都会递归发现包含 `SKILL.md` 的目录。

> 来源：[packages/coding-agent/docs/skills.md — Locations](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md#locations)。

::: checkpoint
全局 skill 放在哪个目录？

--- answer
`~/.pi/agent/skills/` 或 `~/.agents/skills/`。
---
:::

## [m7-practice] 练习：写一个代码审查 Skill

目标：创建一个 skill，指导模型在审查代码时检查：是否有未处理的异常、是否有明显的性能问题、是否有不安全的用户输入。

::: exercise
在 `~/.pi/agent/skills/code-review/SKILL.md` 中写出该 skill。

--- answer
```markdown
---
name: code-review
description: Review code for unhandled exceptions, performance issues, and unsafe user input. Use when the user asks for a code review or submits a diff.
---

# Code Review Skill

When reviewing code, check the following and report findings concisely:

1. **Unhandled exceptions**: Are all async calls awaited? Are errors caught at appropriate boundaries?
2. **Performance**: Are there nested loops over large datasets? Are database queries inside loops?
3. **Unsafe user input**: Is any user input used in shell commands, SQL, or HTML without validation/escaping?

If no issues are found, say so explicitly. If issues are found, include the file path and line number when possible.
```

使用方式：

```bash
/skill:code-review
```

---
:::

## [m7-packages] Pi Packages

Pi Package 是一种分发 extensions、skills、prompt templates、themes 的方式。它可以通过 npm 或 git 安装：

```bash
pi install npm:@foo/bar@1.0.0
pi install git:github.com/user/repo@v1
```

安装后，package 中的 extensions、skills 等资源会被自动发现并加载。

Pi Package 的 `package.json` 可以声明：

- `pi.extensions`：导出 extension 文件；
- `pi.skills`：导出 skill 目录；
- `pi.themes`：导出主题文件；
- `pi.prompts`：导出 prompt templates。

> 来源：[packages/coding-agent/docs/packages.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/packages.md) 与 [packages/coding-agent/docs/extensions.md — Pi Packages](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#pi-packages)。

::: checkpoint
Pi Package 可以通过哪些方式安装？

--- answer
npm 或 git。

:::

## [m7-summary] 模块小结

- Skill 是提示词驱动的能力包，Extension 是运行时行为改造。
- Pi 实现 Agent Skills 标准，但允许 skill 名称与目录名不一致。
- Skill 采用渐进式加载：描述常驻上下文，完整 SKILL.md 按需读取。
- SKILL.md 必需 frontmatter：`name` 和 `description`。
- Skill 可放在全局、项目级、Pi Package 中，也可通过 CLI 临时加载。
- Pi Package 通过 npm/git 分发 extensions、skills、themes、prompts。

> 来源：本地 m6《Extensions：扩展 Pi 的行为》；外部 [packages/coding-agent/docs/skills.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/skills.md)。
