## [m3-handshake] Initialize handshake

The initialize request is the first structured conversation between client and agent. Its job is to establish protocol compatibility and advertise capabilities.

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "capabilities": {},
    "clientInfo": {"name": "minimal-client", "version": "0.1.0"}
  }
}
```

> Verification note: concrete field names and protocol versions can change. Check the current ACP implementation or CLI help before publishing runnable examples.

## [m3-capabilities] Capability negotiation

Capabilities prevent the client and agent from guessing. A client can say what resources, tools, or interaction modes it supports. An agent can say what it can handle.

The important habit is not memorizing a specific capability matrix. The important habit is checking both sides before using an optional feature.

## [m3-failure] Failed negotiation

If the handshake fails, do not continue as if the session is valid. Surface a clear error that says which required capability or protocol version did not match.

## [m3-summary] Module recap

- Initialize creates the first reliable contract between client and agent.
- Capability checks keep optional features honest.
- Exact protocol fields must be verified against the implementation in use.
