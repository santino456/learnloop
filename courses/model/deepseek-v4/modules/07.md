## [m7-ep] 专家并行的通信-计算重叠

MoE 的专家并行（Expert Parallelism, EP）需要大量跨节点通信。DeepSeek-V4 采用**细粒度 EP 方案**，把通信和计算融合成一个流水线内核：

- 每个 MoE 层分为四个阶段：Dispatch（通信）、Linear-1（计算）、Linear-2（计算）、Combine（通信）。
- 把专家切分成多个 wave。
- 当前 wave 计算时，下一个 wave 的 token 传输和已完成 wave 的结果回传可以并行进行。

这样通信延迟被计算隐藏，系统对互联带宽的要求更低。

::: flow
Token -> Dispatch All-to-All -> Linear-1 GEMM -> SwiGLU + FP8 Cast -> Linear-2 GEMM -> Combine All-to-All
:::

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.1 节。

## [m7-tilelang] TileLang 内核开发

为了在开发效率和运行效率之间取得平衡，DeepSeek-V4 使用 **TileLang** 这种领域专用语言（DSL）开发高性能内核。TileLang 允许开发者以接近手写 CUDA 的效率表达算子，同时保持较高的开发速度。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.2 节。

## [m7-deterministic] 批次不变与确定性内核库

训练大模型时，bitwise 可复现性很重要。DeepSeek-V4 提供：

- **Batch-invariant kernel**：不同 batch size 下结果一致。
- **Deterministic kernel library**：训练和推理之间按位可复现。

这对调试训练不稳定、定位精度问题非常关键。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.3 节。

## [m7-fp4] FP4 量化感知训练

DeepSeek-V4 对以下部分使用 **FP4 量化感知训练（QAT）**：

- MoE routed expert 权重
- Indexer 的 QK 路径

这进一步降低了内存和计算开销。论文提到， routed expert 参数使用 FP4 精度；虽然现有硬件上 FP4×FP8 的峰值 FLOPs 与 FP8×FP8 相同，但未来硬件理论上可以更高效。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.4 节与第 1 节。

## [m7-training-framework] 训练框架关键优化

| 优化点 | 作用 |
|--------|------|
| Muon 高效实现 | 混合 ZeRO 策略，降低显存和通信 |
| mHC 高效实现 | 重计算 + 融合内核 |
| 长上下文注意力 | 两阶段上下文并行，管理压缩注意力 |
| 自动微分扩展 | 张量级 checkpoint，细粒度重计算控制 |

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.5 节。

## [m7-inference] 推理框架：KV Cache 与磁盘存储

为了支持 1M token 的高效推理，DeepSeek-V4 设计了：

- **异构 KV Cache 结构**：不同部分使用不同精度和存储策略。
- **On-disk KV Cache 存储**：允许把 KV Cache 放到磁盘，支持共享前缀复用，降低长上下文首次加载成本。

> 来源：[DeepSeek-V4 论文](raw/DeepSeek_V4.pdf)，第 3.6 节。

## [m7-summary] 模块小结

- 细粒度 EP 通过 wave 级流水线隐藏通信延迟。
- TileLang 平衡了内核开发效率与运行效率。
- FP4 QAT、确定性内核、张量级 checkpoint 都是大规模训练可用的关键工程。
- 推理侧的异构 KV Cache 和磁盘存储让 1M token 上下文更实用。
