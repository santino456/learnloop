## [m5-simulator] 世界模型作为环境模拟器

在这个应用中，Qwen-AgentWorld 作为独立的环境模拟器：策略智能体与世界模型是两套模型。模拟器提供两个真实环境难以直接提供的属性：**可扩展性**和**可控性**。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1

## [m5-scaling] 零样本环境扩展

Qwen-AgentWorld 可以在无需领域特定适配的情况下，泛化地模拟大量现实、高度专业化的真实世界环境。

::: evidence
claim: OpenClaw 模拟结果：Qwen-AgentWorld-397B-A17B 模拟了 4000 个 OpenClaw 环境用于智能体 RL，带来 Claw-Eval +4.3 和 QwenClawBench +7.1。用 Qwen3.6-Plus 作为模拟器仅获得边际提升，说明专门的 LWM 训练对构建高保真模拟器至关重要。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.1, Table 6

## [m5-controllable] 可控模拟

可控模拟使用自然语言指令在轨迹级别和 turn 级别塑造 Qwen-AgentWorld 的行为。

::: concept
title: 环境适配（Environment Adaptation）
why: 真实部署往往不够"极端"，可控模拟可以主动制造训练压力。

**环境适配**：通过指令调整模拟器的行为与风格，注入针对性扰动，从而系统性地暴露智能体弱点。控制指令修改模拟风格与内容，产生比真实部署更极端或更有针对性的条件。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2
:::

::: evidence
claim: MCP 可控 Sim RL 结果：没有控制指令的标准 Sim RL 几乎没有收益（Tool Decathlon 甚至从 32.4 降到 31.5）。加入可控模拟后，Tool Decathlon 提升 +3.7，MCPMark 提升 +12.3。可控性不仅是幅度因素，更是该领域 Sim RL 生效的前提。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2, Table 7
status: verified
basis: 论文 §6.1.2 与表 7 的可控 Sim RL 结果。
:::

::: figure
src: assets/qwen-agentworld-table-7.png

## [m5-fictional] 虚构世界构建

::: concept
title: 虚构世界构建（Fictional-World Construction）
why: 防止智能体用参数记忆或真实世界知识绕过模拟环境。

**虚构世界构建**：模拟器生成一个结构上真实、事实上与真实世界完全脱节的虚构环境。以 Search 为例，Qwen-AgentWorld 从一个紧凑的初始设定出发，模拟出完全虚构、自洽的世界。这带来两个结构性优势：
1. 智能体无法绕过搜索工具，用参数记忆直接作答。
2. 智能体不会把训练时的搜索结果与真实世界知识混淆。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2
:::

::: evidence
claim: WideSearch 虚构世界结果：在 Qwen3.5-35B-A3B-SFT 上，可控 Sim RL 把 F1 by Item 从 34.02 提升到 50.31（+16.29），F1 by Row 从 13.72 提升到 24.21（+10.49）。在 Qwen3.5-397B-A17B-SFT 上，Sim RL 仍有 +3.87 和 +6.05 的提升。值得注意的是，训练环境完全虚构：每个搜索结果、网页、事实记录都由 Qwen-AgentWorld 从零发明。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2, Table 8
status: verified
basis: 论文 §6.1.2 与表 8 的 WideSearch 虚构世界结果。
:::

## [m5-sim-vs-real] Sim RL vs. Real RL

::: figure
src: assets/qwen-agentworld-fig-9.png
alt: 图 9：WideSearch 上可控 Sim RL 与 Real RL 在前 60 训练步的对比，展示 F1 分数与平均工具调用频率。
caption: 图 9：WideSearch 上可控 Sim RL 与 Real RL（用真实搜索引擎训练）在前 60 步的对比。两个实验都以 Qwen3.5-35B-A3B-SFT 为基线模型。
:::

::: evidence
claim: Sim RL 超过 Real RL：Sim RL 达到 50.3% F1 by Item，Real RL 为 45.6%。更具信息量的信号来自智能体行为：Sim RL 把 web_extractor 调用从 2.5 提升到 4.0 次/轨迹，而 Real RL 从 2.5 降到 1.5 次。这一差异反映了可控模拟设计：因为模拟搜索摘要故意 withholding 详细内容，Sim-RL 训练出的智能体学会了必须提取完整页面。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2
status: verified
basis: 论文 §6.1.2 与图 9 的 Sim RL vs. Real RL 对比。
:::

::: decision
什么时候用解耦模拟器，什么时候用真实环境？

--- perspective
依据：基于保真度、成本、可扩展性与安全性的权衡。

**使用解耦模拟器（Sim RL）当：**
- 需要扩展到数千个并行环境，而真实基础设施无法支撑。
- 想注入针对性扰动或构建虚构世界做对抗训练。
- 真实环境访问昂贵、缓慢或不可行（不可逆操作、专有部署）。

**使用真实环境（Real RL）当：**
- 需要高保真度，用于安全关键或高风险领域。
- 环境简单快速，扩展不是瓶颈。
- 处于最终验证阶段，需要确认 sim-to-real 迁移。

**重要提醒**：Sim RL 的效果依赖于给世界模型提供足够详细的初始状态；否则模拟保真度下降，下游收益也会衰减。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1
---
:::

## [m5-exercise] 练习：设计一个可控模拟

::: exercise
**任务**：为 Terminal 领域的训练场景设计一条可控模拟指令。

**场景**：你想训练一个智能体，在包安装过程中优雅地处理磁盘空间不足错误。

**要求**：
1. 写出指定初始状态（操作系统、磁盘空间、已安装包）的模拟指令。
2. 指定可控扰动：模拟器应在 pip install 的下载阶段正常完成后，再模拟 "no space left on device" 错误。
3. 指定期望的智能体行为：检测到错误、清理 pip 缓存、并用 `--no-cache-dir` 重试。

--- answer
**验证标准**：模拟指令必须让预期的响应边界足够明确，以减少训练中的幻觉；初始状态必须足够详细，以约束期望观察。

完整答案应包含：
- **初始状态**：Ubuntu 22.04，Python 3.10，pip 23.x，2 GB 可用磁盘，/workspace 已挂载，pandas 未安装。
- **扰动触发**：pip download 完成后、wheel unpack 前，抛出 `OSError: [Errno 28] No space left on device`。
- **期望行为**：捕获错误，执行 `pip cache purge`，然后执行 `pip install --no-cache-dir pandas`。
---
:::

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §6.1.2, Figure 3

---

**模块小结**：作为解耦模拟器，Qwen-AgentWorld 支持零样本环境扩展、带针对性扰动的可控模拟，以及虚构世界构建。在 WideSearch 上，可控 Sim RL 可以超过 Real RL（50.3% vs. 45.6% F1）。初始状态的详细程度是模拟保真度的关键瓶颈。
