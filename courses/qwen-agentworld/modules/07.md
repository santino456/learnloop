## [m7-reasoning] LWM Chain-of-Thought 推理模式

Qwen-AgentWorld 在预测每个环境观察之前都会生成一段 chain-of-thought（思考 trace）。分析 129 条跨四个文本领域的思考 trace 后，可以归纳出三种代表性推理模式。

::: figure
src: assets/qwen-agentworld-fig-12.png
alt: 图 12：Qwen-AgentWorld-397B-A17B 思考 trace 中的代表性 LWM 推理模式：多步因果推理、信息泄露防止、认知边界意识。
caption: 图 12：Qwen-AgentWorld-397B-A17B 思考 trace 中的代表性推理模式。左：Terminal 中的多步因果推理，一条链跨越包管理、进程生命周期、curl 语义和 Python 错误。中：Search 中的信息泄露防止，模型区分智能体知道什么与环境应透露什么。右：Terminal 中的认知边界意识，模型识别计算极限并退回到仅格式输出，而非捏造不可知值。
:::

## [m7-self-correction] 审慎式自我修正

::: concept
title: "Wait!" 自我修正
why: 自我修正把单次生成变成带约束的可满足性搜索。

**"Wait!" 自我修正**：模型用 "Wait!" 作为显式认知中断，重新检查中间预测并在提交前修正。在 129 个 turn 中，共出现 1347 次此类中断（平均每 turn 10.4 次；峰值：单个 SWE turn 56 次）。Terminal 和 MCP 的比率最高（分别为 16.9 和 12.7 次/turn），反映这些领域对状态跟踪的更高要求。

这些自我修正可分为三种功能子类型：
- **事实型**：发现 API 响应格式不正确。
- **认识论型**：认识到上下文内计算的极限（如 `np.random.seed(42)` 例子）。
- **视角采择型**：建模评估者意图或智能体的知识状态。

这种行为把环境预测从单次生成转化为带约束的可满足性搜索。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-leakage] 信息泄露防止

::: concept
title: 信息泄露防止
why: 世界模型必须知道"智能体不该知道什么"，否则训练会泄露答案。

**信息泄露防止**：在 Search 领域，模型掌握一个参考答案，而智能体正在寻找它。当智能体查询与答案无关时，模型会显式防止泄露：识别主题不匹配，确保生成的摘要不会意外暴露目标信息。这是世界模型中的"心智理论"：模型区分自己知道什么，与环境应透露什么。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-epistemic] 认知边界意识

::: concept
title: 认知边界意识
why: 知道"自己不知道"比假装知道更安全、更忠实。

**认知边界意识**：模型识别计算极限，并退回到仅格式输出，而非捏造不可知值。例如，当被要求预测 `np.random.randn(500)` 中超出已知 seed 索引范围的值时，模型记得 `seed(42)` 的前几个值是熟知的，但第 500 个值不可知，因此只输出格式而不捏造具体数字。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.1
:::

## [m7-micro] RL 训练中的微观保真度提升

Turn-level 评估分数捕捉的是聚合质量，但 RL 训练还在更细粒度上提升保真度：单个 URL 标识符、字节级算术、跨 turn API schema 一致性。

::: figure
src: assets/qwen-agentworld-fig-13.png
alt: 图 13：RL 训练中 Search、Terminal、MCP 三个领域的微观保真度提升。
caption: 图 13：RL 训练中的微观保真度提升。上：Search 领域单个样本随 RL 步的演化，URL 标识符、来源多样性、摘要针对性都在改善，尽管它们只占总输出 token 的极小比例。左下：Terminal 领域：模型通过逐字符枚举（包括不可见换行符）完成精确字节级算术。右下：MCP 领域：模型在九个连续 Notion API 调用中保持跨 turn schema 一致性（user ID、parent-child 引用、UUID 格式）。
:::

::: evidence
claim: Search URL 真实性：第 100 步时，模型生成一个 IMDB URL，其标题标识符 `tt2333444` 看似合理但为合成。到第 200 步，标识符变为 `tt2988794`，来源按自然排名出现，摘要包含查询相关的具体事实。URL 标识符在总响应 token 中占比极小，但 RL 仍能提升这些低显著性细节。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.2
status: verified
basis: 论文 §7.2 与图 13 的 Search 微观演化案例。
:::

::: evidence
claim: Terminal 字节级算术：在一个多轮轨迹中，模型 Earlier turn 通过 `cat` 看到文件内容，之后需要预测几轮后的 `wc -c` 输出。模型没有生成一个看似合理的数字，而是在思考 trace 中逐字符枚举（包括不可见的 `\n` 字节），最终通过逐字算术得到精确计数（53 字节）。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.2
status: verified
basis: 论文 §7.2 与图 13 的 Terminal 微观案例。
:::

::: evidence
claim: MCP API schema 保真度：在一个模拟 Notion 工作区的 MCP 轨迹中，模型跨越九个连续 API 调用保持完美一致性：同一 user identifier 出现在每个 `created_by` 字段，每个 block 的 `parent.page_id` 与查询的 block ID 匹配，完整 Notion schema（每个 block 约 20 个字段）无遗漏复现。模型实现了一个有状态的上下文内数据库，在数十个嵌套 JSON 对象中保持引用完整性。
source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7.2
status: verified
basis: 论文 §7.2 与图 13 的 MCP 微观案例。
:::

## [m7-timeline] RL 训练动态

::: timeline
RL 训练动态（Qwen-AgentWorld-35B-A3B，440 步）

- ~0 步：初始状态 —— 五维指标从 SFT 基线出发。
- ~90 步：Format 收敛 —— 达到总提升的 90%。格式约定（JSON 结构、终端提示字符串）是表面模式，RL 快速强化。
- ~250 步：Consistency 收敛 —— 需要更深层跨 turn 状态跟踪能力，逐渐发展。
- 440 步：最终状态 —— Factuality 相对提升最大（+11.3%），但仍是得分最低维度；Quality 提升最小（+0.11）。
:::

::: figure
src: assets/qwen-agentworld-fig-16.png
alt: 图 16：440 个 RL 步的训练动态，展示各维度分数轨迹。
caption: 图 16：440 个 RL 步的训练动态（Qwen-AgentWorld-35B-A3B）。Format 在约 90 步内收敛；Consistency 需要约 250 步。
:::

::: evidence
claim: 维度收敛速度：尽管 RL 奖励是单一标量（五维均值），各维度的改善速度却明显分化。Format 和 Quality 在 90 步内达到总提升的 90%，而 Consistency 和 Realism 需要约 250 步。这说明 RL 优先优化那些在当前训练信号下最容易优化的方面。

## [m7-mapping] 推理模式到下游能力的映射

::: concept
title: 推理到能力的映射
why: 高层次的推理模式需要被学习者对应到可观察的 agent 行为。

**推理到能力的映射**：本模块分析的三种推理模式直接支撑下游智能体能力：
- **审慎自我修正** → 智能体通过内部模拟和修正避免提交不可行动作。
- **信息泄露防止** → 智能体保持自身知识与应从环境交互中推导出的信息之间的干净分离。
- **认知边界意识** → 智能体识别信息不足时退回到安全动作（例如请求澄清或更多上下文）。

Source: [Qwen-AgentWorld](https://arxiv.org/abs/2606.24597), §7
:::

---

**模块小结**：本可选深入模块分析了 Qwen-AgentWorld 如何做出准确预测。三种推理模式——审慎自我修正、信息泄露防止、认知边界意识——从 RL 训练中出现，并直接映射到下游智能体能力。微观保真度提升（URL 真实性、字节级算术、schema 一致性）表明 RL 奖励信号能传播到显式奖励维度之下的粒度。
