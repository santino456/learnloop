## [m6-goal] Goal and setup

The smallest local client starts an agent process, sends an initialize request, sends a prompt, and reads responses.

This module uses pseudocode-style examples. Verify exact method names, fields, and protocol versions against your current agent implementation before running them.

## [m6-process] Start the subprocess

```python
import json
import subprocess

proc = subprocess.Popen(
    ["example-agent", "acp"],
    stdin=subprocess.PIPE,
    stdout=subprocess.PIPE,
    stderr=subprocess.PIPE,
    text=True,
    bufsize=1,
)
```

The command uses `example-agent` instead of a real private setup. Replace it with the agent you are testing.

## [m6-send-receive] Send and receive JSON lines

```python
def send(message):
    proc.stdin.write(json.dumps(message) + "\n")
    proc.stdin.flush()

def receive():
    line = proc.stdout.readline()
    if not line:
        raise RuntimeError("agent closed stdout")
    return json.loads(line)
```

The newline and flush are not cosmetic. Without them the agent may wait forever.

## [m6-prompt] Send a prompt

```python
send({
    "jsonrpc": "2.0",
    "id": 2,
    "method": "prompt",
    "params": {
        "messages": [
            {"role": "user", "content": [{"type": "text", "text": "Explain ACP in one sentence."}]}
        ]
    }
})
print(receive())
```

> Verification note: this is a teaching skeleton. Run the real agent's help command or documentation check before treating it as executable code.

## [m6-buffering] Buffering rules

- Use text mode if the protocol sends text JSON.
- Write one complete JSON object per line.
- Flush after each request.
- Keep stdout clean and send logs to stderr.

## [m6-summary] Module recap

- A minimal local client uses subprocess pipes.
- Line-delimited JSON requires strict newline and flush behavior.
- Runnable examples must be verified against a real agent version.
