## [m5-post-training] 后训练设计

::: concept
title: 三类专家
why: 单一模型难以同时精通代码执行、深度推理和人机交互。

LongCat-2.0 在后训练阶段引入了三类专门化专家组：

- **Agent Experts**：提升复杂真实场景中的自主任务执行能力，聚焦代码、工作流、搜索等垂直领域，优化端到端任务成功率，同时强化工具调用、参数解析、自我修正等原子能力。
- **Reasoning Experts**：增强逻辑推理深度，支持基于问题难度的自适应计算，覆盖数学、STEM、多跳推理等任务。
- **Interaction Experts**：优化人机对齐与用户体验，提升细粒度指令遵循，抑制事实幻觉，并建立安全边界而不损害有用性。

这种"分而治之"的专家设计与 Mistral 的 Granite、DeepSeek 的 reasoning 路线有相似之处，但 LongCat 明确把能力分成 Agent / Reasoning / Interaction 三类，并用 MOPD 做最终融合。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: concept
title: MOPD：多专家融合架构
why: 需要把三类专家的能力合并到一个可部署的模型中。

团队采用 **MOPD** 架构融合三类专家组，使最终模型同时具备：

- 强大的智能体执行能力
- 深度推理能力
- 高质量交互能力

从而能够准确理解复杂用户需求并可靠完成现实世界任务。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

### 关键解读

- **代码 / 智能体任务**：LongCat-2.0 在 Terminal-Bench 2.1 和 SWE-bench Pro 上接近或超过 Gemini 3.1 Pro / GPT-5.5，说明其 Agent Experts 和代码后训练有效。但距离 Opus 4.8 仍有差距，特别是在 SWE-bench Pro（59.5 vs 69.2）。
- **通用智能体**：FORTE 和 RWSearch 表现均衡，但 RWSearch 落后于 GPT-5.5（78.8 vs 85.3），说明复杂搜索场景仍有提升空间。
- **基础能力**：IFEval、GPQA-diamond、IMO-AnswerBench 上落后于 Gemini 3.1 Pro / GPT-5.5，反映 LongCat-2.0 的定位更偏向代码与智能体，而非通用推理。

## [m5-decision] 练习：为场景选模型

::: decision

- 主要任务：仓库级代码迁移与插件重写。
- 上下文：完整代码库 + 迁移文档，可能达到 500K tokens。
- 预算：可接受专用推理集群，但希望单卡显存压力可控。

LongCat-2.0 是否是合适选择？

## [m5-summary] 课程总结

LongCat-2.0 的核心设计可概括为：

1. **1.6T/48B MoE + 1M 上下文**，定位代码与智能体任务。
2. **LSA** 通过 SI/CLI/HI 降低长上下文索引成本。
3. **N-gram Embedding** 在 MoE 正交维度上以 135B 参数提升表示能力。
4. **AI ASIC 超节点 + 6D 并行** 支撑了稳定的大规模训练与高效推理。
5. **Agent / Reasoning / Interaction 三组专家 + MOPD** 实现能力融合。

这些设计共同支撑了 LongCat-2.0 在代码与智能体 benchmark 上的竞争力，但通用推理仍是其相对短板。
