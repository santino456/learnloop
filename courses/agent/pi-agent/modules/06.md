## [m6-what] Extension 是什么

Extension 是 TypeScript 模块，用来扩展 `pi-coding-agent` 的行为。它可以：

- 订阅并拦截 agent 生命周期事件；
- 注册自定义 tool（让 LLM 调用）；
- 注册自定义命令（`/` 命令）；
- 注册自定义 UI 组件和快捷键；
- 持久化状态到 session；
- 自定义 tool 渲染、compaction、权限控制等。

> 来源：[packages/coding-agent/docs/extensions.md](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md)。

::: checkpoint
Extension 不能直接做什么？

A. 注册自定义 tool  
B. 替换 Pi 底层 LLM API 实现  
C. 注册自定义命令  
D. 拦截 tool_call 事件

--- answer
B。Extension 运行在 `pi-coding-agent` 层，不能直接替换 `pi-ai` 底层实现；但它可以注册 provider、自定义 streamFn 等间接影响。
---
:::

## [m6-lifecycle] Extension 生命周期

Extension 导出默认工厂函数，接收 `ExtensionAPI`：

```typescript
import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.on("session_start", async (_event, ctx) => {
    ctx.ui.notify("Extension loaded!", "info");
  });
}
```

Extension 通过 [jiti](https://github.com/unjs/jiti) 加载，因此可以直接写 TypeScript，无需预编译。

常见事件类型包括：

| 事件 | 说明 |
|---|---|
| `session_start` / `session_end` | 会话开始/结束 |
| `resources_discover` | 发现 skills/extensions 等资源 |
| `tool_call` | 工具调用前，可阻断 |
| `tool_result` | 工具结果生成后 |
| `message_*` | 消息生命周期 |
| `agent_*` | agent loop 生命周期 |

> 来源：[packages/coding-agent/docs/extensions.md — Writing an Extension](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#writing-an-extension) 与 [packages/coding-agent/docs/extensions.md — Events](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#events)。

::: checkpoint
Extension 为什么能直接写 TypeScript 而不用 tsc 编译？

--- answer
因为 Pi 使用 jiti 在运行时加载 TypeScript。
---
:::

## [m6-custom-tool] 自定义 Tool

注册一个 tool 需要：

1. 用 TypeBox 定义参数；
2. 提供 `execute` 函数；
3. 返回 `content` 和可选的 `details`。

```typescript
import { Type } from "typebox";

pi.registerTool({
  name: "greet",
  label: "Greet",
  description: "Greet someone by name",
  parameters: Type.Object({
    name: Type.String({ description: "Name to greet" }),
  }),
  async execute(toolCallId, params, signal, onUpdate, ctx) {
    return {
      content: [{ type: "text", text: `Hello, ${params.name}!` }],
      details: {},
    };
  },
});
```

> 来源：[packages/coding-agent/docs/extensions.md — Custom Tools](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#custom-tools)。

::: checkpoint
自定义 tool 的参数类型用什么定义？

--- answer
TypeBox。`typebox` 是 extension 运行时可用的依赖之一。
---
:::

## [m6-custom-command] 自定义命令

```typescript
pi.registerCommand("hello", {
  description: "Say hello",
  handler: async (args, ctx) => {
    ctx.ui.notify(`Hello ${args || "world"}!`, "info");
  },
});
```

注册后，在交互模式下输入 `/hello` 即可触发。

> 来源：[packages/coding-agent/docs/extensions.md — Quick Start](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#quick-start)。

::: checkpoint
注册自定义命令后，用户在交互模式下如何触发？

--- answer
输入 `/命令名`，例如 `/hello`。
---
:::

## [m6-practice] 练习：写一个简单的权限门扩展

目标：拦截所有 `bash` 工具调用，如果命令包含 `rm -rf`，先弹窗确认。

::: exercise
在 `~/.pi/agent/extensions/guard.ts` 中实现该扩展。

--- answer
```typescript
import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.on("tool_call", async (event, ctx) => {
    if (event.toolName === "bash" && event.input.command?.includes("rm -rf")) {
      const ok = await ctx.ui.confirm("Dangerous command", "Allow rm -rf?");
      if (!ok) {
        return { block: true, reason: "Blocked by guard extension" };
      }
    }
  });
}
```

启动时加载：

```bash
pi -e ~/.pi/agent/extensions/guard.ts
```

---
:::

> 来源：[packages/coding-agent/docs/extensions.md — Quick Start](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#quick-start)。

## [m6-ui] 自定义 UI

Extension 可以通过 `ctx.ui` 做多种交互：

- `ctx.ui.notify(message, type)`：通知；
- `ctx.ui.confirm(title, body)`：确认框；
- `ctx.ui.select(title, options)`：选择列表；
- `ctx.ui.input(title, options)`：输入框；
- `ctx.ui.custom(component)`：完整自定义 TUI 组件（基于 pi-tui）。

> 来源：[packages/coding-agent/docs/extensions.md — Custom UI](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#custom-ui)。

::: checkpoint
如果要在 extension 里实现一个复杂的多步骤向导界面，应该使用哪个 API？

--- answer
`ctx.ui.custom()`，它允许你直接用 pi-tui 组件构建完整 UI。
---
:::

## [m6-locations] Extension 加载位置

Pi 自动发现 extension 的位置：

| 位置 | 作用域 |
|---|---|
| `~/.pi/agent/extensions/*.ts` | 全局 |
| `~/.pi/agent/extensions/*/index.ts` | 全局（子目录） |
| `.pi/extensions/*.ts` | 项目级（需信任） |
| `.pi/extensions/*/index.ts` | 项目级（需信任） |
| 通过 `settings.json` 显式配置 | 任意路径或 npm/git 包 |
| CLI `-e ./path.ts` | 一次性测试 |

注意：只有放在自动发现位置的 extension 才能通过 `/reload` 热重载；`-e` 路径仅用于临时测试。

> 来源：[packages/coding-agent/docs/extensions.md — Extension Locations](https://github.com/earendil-works/pi/blob/371adcf37130629ffb9bbeed9f5548ce08ffa93b/packages/coding-agent/docs/extensions.md#extension-locations)。

::: checkpoint
哪种 extension 加载方式支持 `/reload` 热重载？

--- answer
放在 `~/.pi/agent/extensions/` 或 `.pi/extensions/` 自动发现位置的 extension。
---
:::

## [m6-summary] 模块小结

- Extension 是 TypeScript 模块，通过默认工厂函数接收 `ExtensionAPI`。
- 可以注册自定义 tool、command、shortcut、flag、provider、UI。
- 通过事件机制可以拦截 tool_call、修改消息、持久化状态。
- 常用 `ctx.ui` 做用户交互；复杂 UI 用 `ctx.ui.custom()` 基于 pi-tui 实现。
- 自动发现位置支持 `/reload` 热重载；`-e` 仅用于临时测试。
