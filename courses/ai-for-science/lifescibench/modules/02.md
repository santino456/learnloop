## [m2-workflows] 七大科学工作流

LifeSciBench 把任务按科研活动类型分为七个 workflow：

| 工作流 | 含义 |
|--------|------|
| Evidence handling | 从论文、图表、监管文件、实验输出或结构化数据中提取、解释、比较或综合科学证据 |
| Analysis | 对生物、化学或实验数据进行定量、统计、计算或机制分析 |
| Design and optimization | 设计或优化分子、实验、构建体、方案或筛选策略 |
| Scientific reasoning | 解释机制、评估假设、识别因果关系或在不确定性下推理 |
| Validation and operations | 故障排查、质控、可行性评估、运营规划或转化风险评估 |
| Translation | 把临床前或生物学证据连接到临床相关性、治疗意义、患者影响、试验设计、安全性等 |
| Scientific communication | 为特定受众或决策场景总结、改写、解释科学发现 |

> 来源：LifeSciBench preprint Appendix B.1 Workflow Taxonomy。

::: checkpoint

A. Scientific communication  
B. Analysis  
C. Translation  
D. Validation and operations

--- answer
B。该任务的核心是对生物数据进行计算分析与解读，属于 Analysis workflow。示例见 preprint Section D.1。

:::

## [m2-domains] 七大生物与科学领域

除了 workflow，任务还按领域分层抽样：

| 领域 | 含义 |
|------|------|
| Genomics | 基因组变异、基因调控、测序解读、基因编辑、构建体设计、遗传机制 |
| Chemistry / MedChem | 小分子、药物化学、构效关系、化学表征、ADME、优化权衡 |
| Protein + Structural Biology | 蛋白序列、结构、稳定性、折叠、结合、工程化与结构解读 |
| Molecular + Cell Biology | 细胞机制、通路、分子生物学实验、扰动与生物学解读 |
| Assays + Screening | 实验设计、筛选读数、对照、验证、故障排查与实验决策 |
| Bioinformatics / Comp Bio | 生物数据的计算分析、组学流程、统计推断、可视化与工具 |
| Clinical / Translational Science | 临床前到临床推理、生物标志物、试验设计、患者影响、转化风险 |

> 来源：LifeSciBench preprint Appendix B.2 Bio Domain Taxonomy。

::: checkpoint

D. Clinical / Translational Science

B。Golden Gate 克隆属于分子生物学构建体设计，主要涉及 Genomics 领域。示例见 preprint Section D.2。

## [m2-process] 任务编写与审查流程

LifeSciBench 的任务不是简单从网上抓取，而是经过多轮专家流程：

1. **专家作者**（具有博士或产业经验的生命科学家）根据真实工作经验编写任务和 rubric。
2. **内部审查**：检查科学性、可行性、与 rubric 的一致性。
3. **外部专家验证**：独立评审任务的真实世界相关性、科学推理、领域匹配度和整体有用性。
4. **修订与定稿**：根据反馈修改任务和评分标准。

最终数据集包含 **750 道任务**、**19,020 条 rubric 标准**，覆盖 7 个 workflow 和 7 个领域。

> 来源：LifeSciBench preprint Section 4 Dataset Creation 与 Figure 3。

## [m2-composition] 数据构成

任务在 workflow 和 domain 上大致均衡分布：

- Analysis 与 Design, Optimization & Prediction 是最多的两类。
- Evidence handling 与 Scientific reasoning 也占较大比例。
- Translation 与 Scientific communication 相对较少。
- 约 **54%** 的任务附带 artifact（原始数据、图片、序列、表格等）。

> 来源：LifeSciBench preprint Section 4.5 Dataset Composition 与 Figure 13。

::: checkpoint

## [m2-summary] 模块小结

- LifeSciBench 用 7 个 workflow × 7 个 domain 的矩阵覆盖真实科研活动。
- 任务由专家编写，经多轮审查与外部验证。
- 750 道题中超过一半附带 artifact，评分依赖 19,020 条 rubric 细则。

::: checkpoint
LifeSciBench 的任务编写流程中，哪一步用于确认任务在真实科研中有用且科学可靠？

A. 模型评估  
B. 外部专家验证  
C. 自动从网上抓取  
D. 学生试做

--- answer
B。外部专家验证（external expert validation）专门评估任务的真实世界相关性、科学推理质量和领域匹配度。
