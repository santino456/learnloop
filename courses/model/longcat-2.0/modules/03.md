## [m3-motivation] 为什么还需要 N-gram Embedding

::: concept
title: MoE 的稀疏度已经很高
why: 当稀疏度达到一定程度后，继续加专家收益递减。

LongCat-2.0 继承自 [LongCat-Flash-Lite](https://www.longcatai.org/models/flash-lite) 的 N-gram Embedding，其目标是在与 MoE 正交的稀疏维度上扩展参数，从而提高参数利用效率。报告中给出的判断是：

- 在不考虑 N-gram Embedding 的情况下，模型稀疏度已经约 **97%**。
- 继续用同等规模（135B）的标准专家扩展，收益很小。
- 但把同等参数放到 N-gram Embedding 上，收益远超标准专家。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: concept
title: N-gram Embedding 的实现
why: 了解 n-gram size 与参数规模的具体配置。

为匹配 LongCat-2.0 的超大模型规模，N-gram Embedding 的配置如下：

- **n-gram size = 5**
- **135B 参数**
- 通过 n-gram token 组合把 embedding 空间扩大约 **100 倍**
- 捕获更丰富的局部上下文，强化 token 级表示

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: concept
title: 两条扩展原则
why: 任何额外参数都需要控制比例，否则边际收益会下降。

报告提出了两条约束原则：

1. **MoE 稀疏度已过甜点**：继续堆专家收益低，因此把参数投到 N-gram Embedding 更划算。
2. **N-gram Embedding 比例必须控制在最优区间**：当 N-gram 参数超过总参数 50% 时，相对加专家的优势会减弱。LongCat-2.0 将其严格控制在 **10% 以内**。

Source: [LongCat-2.0 官方博客](https://longcat.chat/blog/longcat-2.0/)
:::

::: compare
left: 继续扩展标准 MoE 专家
right: 增加 N-gram Embedding

- 稀疏维度 | 与现有专家共享同一稀疏空间 | 在正交的 n-gram 维度上扩展
- 边际收益 | 稀疏度 97% 后收益很小 | 同等 135B 参数收益更高
- 推理影响 | 增加大 batch 解码的 I/O | 把参数从 expert 移出，降低大 batch I/O
- 比例上限 | 无硬性上限 | 建议控制在总参数 10% 以内
:::

## [m3-exercise] 练习：判断扩展策略

::: exercise
假设你要为一个已有 96% 稀疏度的 MoE 模型增加 100B 参数，你会优先加专家还是加 N-gram Embedding？给出理由。

--- answer
优先加 N-gram Embedding，因为 MoE 稀疏度已经很高，继续加专家的边际收益低；而 N-gram Embedding 在正交稀疏维度上扩展，能在控制比例的前提下获得更高收益，并减少推理时 expert I/O。
---
:::
